# คู่มือเริ่มต้นใช้งาน Novel Writer Skills

ยินดีต้อนรับสู่ **Novel Writer Skills** - เครื่องมือช่วยเขียนนิยายด้วย AI ที่ออกแบบมาสำหรับ Claude Code โดยเฉพาะ!

## เริ่มต้นใช้งานอย่างรวดเร็ว

### 1. การติดตั้ง

```bash
npm install -g novel-writer-skills
```

### 2. สร้างโปรเจกต์แรกของคุณ

```bash
# สร้างโปรเจกต์ใหม่
novelwrite init my-first-novel

# เข้าไปในไดเรกทอรีโปรเจกต์
cd my-first-novel
```

### 3. เปิดใน Claude Code

เปิดโฟลเดอร์โปรเจกต์ใน Claude Code คุณจะเห็น:

```
my-first-novel/
├── .claude/
│   ├── commands/      # 13 คำสั่งสแลช
│   └── skills/        # 7 Agent Skills
├── .specify/
├── stories/
└── spec/
```

## ขั้นตอนการสร้างสรรค์ 7 ขั้นตอน

### ขั้นตอนที่ 1: สร้างรัฐธรรมนูญการเขียน

พิมพ์ใน Claude Code:

```
/constitution
```

คำสั่งนี้จะนำทางคุณในการกำหนด:
- ✅ หลักการสร้างสรรค์หลัก
- ✅ มาตรฐานคุณภาพ
- ✅ ความชอบด้านรูปแบบ
- ✅ ข้อกำหนดด้านเนื้อหา

**ระยะเวลาโดยประมาณ**: 15-20 นาที

### ขั้นตอนที่ 2: กำหนดข้อกำหนดของเรื่อง

```
/specify
```

ระบุรายละเอียดเรื่องราวของคุณ:
- 📖 สรุปหนึ่งประโยค
- 👥 ผู้อ่านกลุ่มเป้าหมาย
- ⚔️ ความขัดแย้งหลัก
- 👤 ตัวละครหลัก
- 🎯 เกณฑ์ความสำเร็จ

**ระยะเวลาโดยประมาณ**: 30-45 นาที

### ขั้นตอนที่ 3: ชี้แจงประเด็นที่คลุมเครือ

```
/clarify
```

AI จะช่วยคุณผ่านคำถามสำคัญ 5 ข้อ:
- ❓ ระบุจุดที่คลุมเครือในข้อกำหนด
- 💡 ตัดสินใจอย่างชัดเจน
- 📝 อัปเดตเอกสารข้อกำหนดโดยอัตโนมัติ

**ระยะเวลาโดยประมาณ**: 10-15 นาที

### ขั้นตอนที่ 4: จัดทำแผนการเขียน

```
/plan
```

ออกแบบแผนการดำเนินการ:
- 📚 โครงสร้างบท
- 📈 การกระจายจังหวะ
- 🎭 เส้นทางการพัฒนาตัวละคร
- 🔮 แผนการวางปม

**ระยะเวลาโดยประมาณ**: 45-60 นาที

### ขั้นตอนที่ 5: แบ่งรายการงาน

```
/tasks
```

สร้างงานที่ดำเนินการได้:
- ✅ จัดเรียงตามลำดับความสำคัญ
- 🔗 ระบุความสัมพันธ์ของการพึ่งพา
- ⏱️ ประเมินปริมาณงาน

**ระยะเวลาโดยประมาณ**: 20-30 นาที

### ขั้นตอนที่ 6: เริ่มเขียน

```
/write
```

AI ช่วยในการสร้างสรรค์:
- 🤖 สร้างเนื้อหาตามข้อกำหนดและแผน
- 🎨 ประยุกต์ใช้ความรู้ประเภทเรื่องโดยอัตโนมัติ
- 🔍 ตรวจสอบความสอดคล้องเบื้องหลัง
- ⚡ ประยุกต์ใช้เทคนิคการเขียนแบบเรียลไทม์

**จังหวะที่แนะนำ**: เขียนครั้งละ 1-2 บาท ทุก 3-5 บาทให้หยุดวิเคราะห์

### ขั้นตอนที่ 7: ยืนยันคุณภาพ

```
/analyze
```

ตรวจสอบคุณภาพอย่างครอบคลุม:
- ✅ การปฏิบัติตามรัฐธรรมนูญ
- ✅ ความครบถ้วนตามข้อกำหนด
- ✅ ความสอดคล้องของเนื้อหา
- ✅ การบรรลุมาตรฐานคุณภาพ

**ความถี่ที่แนะนำ**: รันทุก 5 บาท

## Agent Skills เปิดใช้งานอัตโนมัติ

### ไม่ต้องเรียกใช้ด้วยตนเอง

เมื่อคุณกำลังสร้างสรรค์ Skills ที่เกี่ยวข้องจะ **เปิดใช้งานอัตโนมัติ**:

**ความรู้ประเภทเรื่อง**:
- พูดถึง "รักหวานแหวว" → Romance Skill เปิดใช้งาน
- พูดถึง "สืบสวนสอบสวน" → Mystery Skill เปิดใช้งาน
- พูดถึง "แฟนตาซี" → Fantasy Skill เปิดใช้งาน

**เทคนิคการเขียน**:
- เขียนบทสนทนา → Dialogue Techniques เปิดใช้งาน
- เขียนฉาก → Scene Structure เปิดใช้งาน

**การรับประกันคุณภาพ**:
- ระหว่างการเขียน → Consistency Checker ตรวจสอบเบื้องหลัง
- ตลอดกระบวนการ → Workflow Guide ทำงานอยู่ตลอด

### การแจ้งเตือนเชิงรุก

Skills จะแจ้งเตือนเมื่อตรวจพบปัญหา:

```
⚠️ การแจ้งเตือนการตรวจสอบความสอดคล้อง

ปัญหา: ลักษณะตัวละครไม่ตรงกัน
ตำแหน่ง: บทที่ 5 วรรคที่ 3

ข้อความปัจจุบัน: "ดวงตาสีเขียวของแมรี่..."
ลักษณะที่กำหนดไว้: "สีตา: น้ำเงิน"

คุณต้องการให้ฉันแก้ไขหรือไม่?
```

## คำสั่งติดตามและยืนยัน

### เริ่มต้นระบบติดตาม

```
/track-init
```

ใช้งานครั้งแรก สร้างไฟล์ติดตาม

### การติดตามแบบครบวงจร

```
/track
```

รันหลังจากเขียนแต่ละบทเสร็จ อัปเดต:
- 📊 การติดตามเนื้อเรื่อง
- ⏰ เส้นเวลา
- 👥 ความสัมพันธ์ของตัวละคร
- 🌍 สถานะโลก

### การตรวจสอบเฉพาะด้าน

```
/plot-check   # ความสอดคล้องของเนื้อเรื่อง
/timeline     # การจัดการเส้นเวลา
/relations    # ความสัมพันธ์ตัวละคร
/world-check  # การยืนยันโลก
```

## ระบบปลั๊กอิน

### ดูปลั๊กอินที่ติดตั้ง

```bash
novelwrite plugin:list
```

### ติดตั้งปลั๊กอิน

```bash
novelwrite plugin:add authentic-voice
```

### ถอดปลั๊กอิน

```bash
novelwrite plugin:remove authentic-voice
```

### ปลั๊กอินอย่างเป็นทางการ

- **authentic-voice**: ปลั๊กอินการเขียนด้วยน้ำเสียงมนุษย์จริง เพิ่มความเป็นต้นฉบับ

## คำถามที่พบบ่อย

### Q: ฉันสามารถข้ามบางขั้นตอนได้หรือไม่?

A: ได้ แต่บางคำสั่งพึ่งพากัน:
- `/write` ต้องมี `/specify` และ `/plan`
- ขั้นตอนขั้นต่ำ: `/constitution` → `/specify` → `/write`

### Q: Skills จะรบกวนการสร้างสรรค์ของฉันหรือไม่?

A: ไม่! Skills เป็น **แบบรับฟัง**:
- เปิดใช้งานเฉพาะเมื่อเกี่ยวข้อง
- ให้คำแนะนำไม่ใช่การบังคับ
- คุณมีสิทธิ์ตัดสินใจขั้นสุดท้ายเสมอ

### Q: จะปรับระดับความเข้มงวดของการตรวจสอบความสอดคล้องได้อย่างไร?

A: บอก AI ในการสนทนา:
```
"กรุณาใช้โหมดยืดหยุ่นในการตรวจสอบความสอดคล้อง
เพราะนี่คือนิยายแฟนตาซี"
```

### Q: ฉันมีโครงเรื่องอยู่แล้ว จะทำอย่างไร?

A: ใช้ `/specify` เพื่อแปลงโครงเรื่องที่มีอยู่เป็นรูปแบบ novelwrite
แล้วดำเนินการในขั้นตอนถัดไป
