# เอกสารความต้องการผลิตภัณฑ์ Novel Writer Skills

**เวอร์ชัน**: v1.0  
**วันที่**: 2025-10-18  
**สถานะ**: ร่าง

---

## 1. การวางตำแหน่งผลิตภัณฑ์

### 1.1 ภาพรวมโครงการ

**novel-writer-skills** เป็นเครื่องมือช่วยเขียนนิยายด้วย AI ที่ออกแบบมาโดยเฉพาะสำหรับ **Claude Code** โดยผสานรวมกับระบบ Slash Commands และ Agent Skills ของ Claude อย่างลึกซึ้ง

- **เทคโนโลยี**: เฉพาะ Claude Code
- **ความสามารถหลัก**: ระเบียบวิธีเจ็ดขั้นตอน + ผู้ช่วยอัจฉริยะ Agent Skills
- **กลุ่มเป้าหมาย**: ผู้เขียนที่ใช้ Claude Code ในการเขียนนิยาย

### 1.2 ความสัมพันธ์กับ novel-writer

- **novel-writer**: ข้ามแพลตฟอร์ม (เครื่องมือ AI 13 ชนิด) ระเบียบวิธีพื้นฐาน
- **novel-writer-skills**: เฉพาะ Claude Code, เพิ่มประสิทธิภาพเชิงลึก

**สิ่งที่ใช้ร่วมกัน**: ระเบียบวิธีเจ็ดขั้นตอน, โครงสร้างไฟล์, ระบบติดตาม
**ความแตกต่าง**: เลิกสนับสนุนข้ามแพลตฟอร์ม, เพิ่มระบบ Agent Skills อัจฉริยะ

## 3. ฟังก์ชันหลัก

### 3.1 Slash Commands (ผู้ใช้เรียกใช้โดยตรง)

#### คำสั่งระเบียบวิธีเจ็ดขั้นตอน

| คำสั่ง | หน้าที่ | ผลลัพธ์ |
|------|------|------|
| `/constitution` | สร้างรัฐธรรมนูญการเขียน | `.specify/memory/constitution.md` |
| `/specify` | กำหนดข้อกำหนดของเรื่อง | `stories/[name]/specification.md` |
| `/clarify` | ชี้แจงประเด็นที่คลุมเครือ (5 คำถาม) | อัปเดต specification.md |
| `/plan` | จัดทำแผนการเขียน | `stories/[name]/creative-plan.md` |
| `/tasks` | แบ่งรายการงาน | `stories/[name]/tasks.md` |
| `/write` | ดำเนินการเขียนบท | `stories/[name]/content/chapter-XX.md` |
| `/analyze` | วิเคราะห์ยืนยันคุณภาพ | รายงานการวิเคราะห์ (สองโหมด: กรอบ/เนื้อหา) |

#### คำสั่งติดตามและยืนยัน

| คำสั่ง | หน้าที่ |
|------|------|
| `/track-init` | เริ่มต้นระบบติดตาม |
| `/track` | อัปเดตการติดตามแบบครบวงจร |
| `/plot-check` | ตรวจสอบความสอดคล้องของเนื้อเรื่อง |
| `/timeline` | การจัดการเส้นเวลา |
| `/relations` | ติดตามความสัมพันธ์ของตัวละคร |
| `/world-check` | ยืนยันความสอดคล้องของโลก |
| `/checklist` | รายการตรวจสอบคุณภาพ |

### 3.2 Agent Skills (AI เปิดใช้งานอัตโนมัติ)

#### หลักการออกแบบ Skills

- **เปิดใช้งานแบบรับฟัง**: AI ตัดสินใจตามบริบทโดยอัตโนมัติ
- **ไม่รับรู้**: ผู้ใช้ไม่ต้องเรียกใช้ด้วยตนเอง
- **ประยุกต์ใช้ต่อเนื่อง**: ทำงานตลอดการสนทนา

#### การแบ่งหมวดหมู่ Skills

**1. Genre Knowledge Skills (ฐานความรู้ประเภทเรื่อง)**

ให้ข้อมูลรูปแบบและเทคนิคการเขียนตามประเภทของนิยายโดยอัตโนมัติ:

- `romance.md` - รูปแบบนิยายรัก
- `mystery.md` - เทคนิคนิยายสืบสวนสอบสวน
- `fantasy.md` - ข้อกำหนดการสร้างโลกแฟนตาซี
- `sci-fi.md` - การสร้างโลกนิยายวิทยาศาสตร์
- `thriller.md` - การควบคุมจังหวะนิยายระทึกขวัญ

**ตัวอย่างการกระตุ้น**: ผู้ใช้พูดว่า "ฉันจะเขียนนิยายรัก" → romance skill เปิดใช้งานอัตโนมัติ

**2. Writing Techniques Skills (เทคนิคการเขียน)**

ประยุกต์ใช้แนวทางปฏิบัติที่ดีที่สุดในสถานการณ์การเขียนเฉพาะ:

- `dialogue-techniques.md` - ความเป็นธรรมชาติของบทสนทนา
- `scene-structure.md` - การสร้างฉาก
- `character-arc.md` - เส้นทางการพัฒนาตัวละคร
- `pacing-control.md` - การควบคุมจังหวะ
- `description-depth.md` - ระดับรายละเอียดในการบรรยาย

**ตัวอย่างการกระตุ้น**: ขณะเขียนฉากบทสนทนา → dialogue-techniques เปิดใช้งานอัตโนมัติ

**3. Quality Assurance Skills (การตรวจสอบอัจฉริยะ)**

ตรวจสอบและแจ้งเตือนอัตโนมัติระหว่างการเขียน:

- `consistency-checker.md` - การตรวจสอบความสอดคล้อง (ตัวละคร โลก เส้นเวลา)
- `pov-validator.md` - การตรวจสอบมุมมอง
- `continuity-tracker.md` - การติดตามความต่อเนื่อง
- `pacing-monitor.md` - การตรวจสอบจังหวะ

**ตัวอย่างการกระตุ้น**: ตรวจพบความขัดแย้งระหว่างการเขียน → แจ้งเตือนอัตโนมัติ

### 3.3 การทำงานร่วมกันของ Skills และ Commands

```
ผู้ใช้: ฉันจะเขียนนิยายรัก

[Skills เปิดใช้งาน]
✓ romance-novel-conventions (ความรู้ประเภทเรื่อง)
✓ workflow-guide (แนะนำการใช้ระเบียบวิธีเจ็ดขั้นตอน)

AI ตอบกลับ:
"เยี่ยม! ใช้แนวทางที่เป็นระบบในการสร้างสรรค์กันเถอะ 
เริ่มด้วยการรัน /constitution เพื่อกำหนดหลักการเขียนของคุณ
จากนั้น /specify เพื่อระบุข้อกำหนดของเรื่อง..."

[ในการเขียนขั้นต่อไป]
✓ รัน /write → dialogue-techniques เปิดใช้งานอัตโนมัติ
✓ ระหว่างการเขียน → consistency-checker ตรวจสอบเบื้องหลัง
✓ ตรวจพบปัญหา → แจ้งเตือนผู้ใช้เชิงรุก
```

## 4. ข้อกำหนดทางเทคนิค

### 4.1 รูปแบบ Slash Command

```markdown
---
description: คำอธิบายสั้นของคำสั่ง (หนึ่งประโยค)
---

# ชื่อคำสั่ง

## เป้าหมาย
[สิ่งที่คำสั่งนี้ต้องการทำให้สำเร็จ]

## ขั้นตอน
[คำอธิบายขั้นตอน]

## ผลลัพธ์
[สร้างไฟล์อะไร]

## ตัวอย่าง
[ตัวอย่างการใช้งาน]
```

### 4.2 รูปแบบ Agent Skill

```yaml
---
name: skill-identifier
description: "Use when [เงื่อนไขการกระตุ้น] - [คำอธิบายฟังก์ชัน]"
allowed-tools: Read, Grep, Glob
---

# ชื่อ Skill

## Quick Reference
[ตารางอ้างอิงด่วน]

## Core Concepts
[แนวคิดหลัก]

## Best Practices
[แนวทางปฏิบัติที่ดีที่สุด]

## Common Pitfalls
[ข้อผิดพลาดที่พบบ่อย]
```

**ประเด็นสำคัญในการเขียน Description**:

- ต้องมีเงื่อนไขการกระตุ้นที่ชัดเจน
- อธิบายว่ามีคุณค่าอะไรบ้าง
- ตัวอย่าง: `"Use when user mentions romance or love story - provides genre conventions and emotional beat planning for romance writing"`

## 5. เครื่องมือ CLI

### 5.1 คำสั่งหลัก

```bash
# ติดตั้ง
npm install -g novel-writer-skills

# เริ่มต้นโปรเจกต์
novelwrite init my-novel

# ติดตั้งปลั๊กอิน
novelwrite plugin:add authentic-voice

# แสดงรายการปลั๊กอิน
novelwrite plugin:list

# อัปเกรดโปรเจกต์
novelwrite upgrade

# ตรวจสอบสถานะ
novelwrite check
```

### 5.2 ขั้นตอนการเริ่มต้น

```bash
novelwrite init my-novel
```

การดำเนินการ:
1. สร้างโครงสร้างไดเรกทอรีโปรเจกต์
2. คัดลอก `.claude/commands/` และ `.claude/skills/`
3. เริ่มต้นการตั้งค่า `.specify/`
4. สร้างเทมเพลต `spec/tracking/`
5. สร้าง README.md

## 6. แผนงานการพัฒนา

### 6.1 MVP (4-6 สัปดาห์)

**เป้าหมาย**: ทดสอบความสามารถในการใช้งานของฟังก์ชันหลัก

**สิ่งที่จัดส่ง**:
- ✅ Commands ระเบียบวิธีเจ็ดขั้นตอน (7 คำสั่ง)
- ✅ Commands ติดตามและยืนยัน (6 คำสั่ง)
- ✅ Genre Knowledge Skills 2-3 ชนิด (romance, mystery, fantasy)
- ✅ Writing Techniques Skills 2-3 ชนิด (dialogue, scene-structure)
- ✅ Quality Assurance Skill 1 ชนิด (consistency-checker)
- ✅ เครื่องมือ CLI พื้นฐาน (init, plugin)
- ✅ เอกสารหลัก

**เกณฑ์ความสำเร็จ**:
- Commands สามารถดำเนินการตามกระบวนการเจ็ดขั้นตอนได้ถูกต้อง
- Skills เปิดใช้งานในสถานการณ์ที่ถูกต้อง (อัตราการเปิดใช้งาน > 80%)
- ผลตอบรับจากผู้ใช้ทดสอบ 5-10 รายเป็นบวก

### 6.2 Phase 2 (6-8 สัปดาห์)

**เป้าหมาย**: ฟังก์ชันครบถ้วนและระบบนิเวศปลั๊กอิน

**สิ่งที่จัดส่ง**:
- ✅ Genre Skills ครบทุกประเภท (5 ประเภท)
- ✅ Writing Skills ครบทุกประเภท (6 เทคนิค)
- ✅ QA Skills ครบทุกประเภท (4 การตรวจสอบ)
- ✅ ปรับปรุงระบบปลั๊กอิน
- ✅ ปลั๊กอินอย่างเป็นทางการ (authentic-voice, translate ฯลฯ)
- ✅ เอกสารการพัฒนาปลั๊กอิน

**เกณฑ์ความสำเร็จ**:
- Skills ครอบคลุมสถานการณ์การสร้างสรรค์หลัก
- ความแม่นยำของการตรวจสอบความสอดคล้อง > 85%
- ชุมชนเริ่มมีส่วนร่วมในการพัฒนาปลั๊กอิน

### 6.3 Phase 3 (8-10 สัปดาห์)

**เป้าหมาย**: การปรับปรุงประสิทธิภาพและการส่งเสริมการตลาด

**สิ่งที่จัดส่ง**:
- ✅ การปรับปรุงประสิทธิภาพ (โหลด Skills < 2 วินาที)
- ✅ Commands ขั้นสูง (polish-prose, theme-analysis ฯลฯ)
- ✅ โปรเจกต์ตัวอย่างสมบูรณ์
- ✅ วิดีโอสอน
- ✅ การสร้างชุมชน

**เกณฑ์ความสำเร็จ**:
- ผู้ใช้ที่ใช้งานจริง 100+ คน
- อัตราการแจ้งเตือนผิดพลาด < 10%
- GitHub Stars > 200

| ความเสี่ยง | ผลกระทบ | แนวทางรับมือ |
|------|------|------|
| การเปิดใช้งาน Skills ไม่แม่นยำ | ประสบการณ์ผู้ใช้ไม่ดี | เขียน description อย่างระมัดระวัง ทดสอบอย่างเพียงพอ |
| อัตราการแจ้งเตือนผิดพลาดสูง | ความน่าเชื่อถือลดลง | แบ่งระดับการแจ้งเตือน (Critical/Warning/Note) |
| ปัญหาด้านประสิทธิภาพ | โหลดช้า กระทบการใช้งาน | โหลดแบบเลื่อน (Lazy loading), ปรับขนาด Skill |

### 8.2 ความเสี่ยงด้านผลิตภัณฑ์

| ความเสี่ยง | ผลกระทบ | แนวทางรับมือ |
|------|------|------|
| รองรับเฉพาะ Claude ฐานผู้ใช้เล็ก | การเติบโตจำกัด | มุ่งเน้นความลึกแทนความกว้าง สร้างประสบการณ์ที่ดีที่สุด |
| เส้นทางการเรียนรู้สูง | ผู้ใช้ใหม่ออกจากระบบ | ปรับปรุงเอกสาร ให้ตัวอย่าง คำแนะนำแบบขั้นตอน |
| การมีส่วนร่วมของชุมชนต่ำ | การพัฒนาระบบนิเวศช้า | กลไกสร้างแรงจูงใจ ลดอุปสรรคในการมีส่วนร่วม |

## 9. ขั้นตอนการดำเนินการต่อไป

### 9.1 ดำเนินการทันที (สัปดาห์นี้)

1. **สร้างโครงสร้างโปรเจกต์**
   - สร้างที่เก็บ `novel-writer-skills`
   - ตั้งค่าโครงสร้างโปรเจกต์
   - กำหนดค่า TypeScript และเครื่องมือสร้าง

2. **พัฒนา CLI พื้นฐาน**
   - คำสั่ง `novel-skills init`
   - ไฟล์เทมเพลตโปรเจกต์

3. **เขียน Command แรก**
   - คำสั่ง `/constitution`
   - ทดสอบการทำงานใน Claude Code

### 9.2 เป้าหมายระยะใกล้ (ภายใน 2 สัปดาห์)

1. **พัฒนา Commands ระเบียบวิธีเจ็ดขั้นตอนให้ครบ**
   - คำสั่งหลัก 7 คำสั่งทั้งหมด
   - เขียนเอกสารการใช้งาน

2. **พัฒนา Skills พื้นฐาน 2-3 ชนิด**
   - romance-novel-conventions
   - dialogue-techniques
   - consistency-checker

3. **ทดสอบและปรับปรุง**
   - เชิญผู้ใช้ทดสอบ 5-10 ราย
   - รวบรวมผลตอบรับ ปรับปรุงอย่างรวดเร็ว

### 9.3 เป้าหมายระยะกลาง (4-6 สัปดาห์)

1. **พัฒนา MVP ให้เสร็จสมบูรณ์**
   - ฟังก์ชันหลักทั้งหมด
   - เอกสารครบถ้วน
   - โปรเจกต์ตัวอย่าง

2. **เตรียมพร้อมสำหรับการเปิดตัว**
   - เผยแพร่แพ็กเกจ npm
   - เปิดที่เก็บ GitHub แบบสาธารณะ
   - เขียนประกาศการเปิดตัว
