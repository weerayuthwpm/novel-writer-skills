# คู่มือการใช้ Agent Skills

Agent Skills คือหัวใจหลักของ Novel Writer Skills - พวกมันจะ **เปิดใช้งานอัตโนมัติ** เมื่อคุณกำลังสร้างสรรค์ ไม่ต้องเรียกใช้ด้วยตนเอง

## หลักการทำงานของ Skills

### การเปิดใช้งานอัตโนมัติ

เมื่อคุณกำลังสร้างสรรค์ใน Claude Code AI จะตัดสินใจเปิดใช้งาน Skills ใดบ้างโดยอัตโนมัติตามบริบท:

```
คุณพูด: "ฉันจะเขียนนิยายรัก"
     ↓
[เปิดใช้งาน Romance Skill อัตโนมัติ]
     ↓
AI ตอบกลับโดยประยุกต์ใช้รูปแบบการเขียนนิยายรักโดยอัตโนมัติ
```

### การประยุกต์ใช้ต่อเนื่อง

เมื่อเปิดใช้งานแล้ว Skills จะทำงานตลอดการสนทนา:
- ทำงานโดยไม่รับรู้
- ให้คำแนะนำเชิงวิชาชีพอย่างต่อเนื่อง
- แจ้งเตือนเชิงรุกเมื่อตรวจพบปัญหา

## การแบ่งหมวดหมู่ Skills

### 1. Genre Knowledge Skills (ฐานความรู้ประเภทเรื่อง)

#### Romance (นิยายรัก)

**เงื่อนไขการเปิดใช้งาน**:
- พูดถึง "นิยายรัก" "เรื่องราวความรัก" "ความรัก"
- ข้อกำหนดของเรื่องมีองค์ประกอบของความรัก

**สิ่งที่ให้**:
- จังหวะอารมณ์ (พบกันครั้งแรก จูบครั้งแรก ช่วงมืดมน ตอนจบ)
- คู่มือจังหวะ (ค่อยเป็นค่อยไป vs เร่าร้อน)
- กับดักที่พบบ่อย (รักแรกพบ ความเข้าใจผิดเป็นความขัดแย้งเดียว)
- ข้อกำหนดตอนจบแบบ HEA/HFN

**เวลาที่ประยุกต์ใช้**:
- `/specify` - แนะนำให้รวมส่วนโค้งความสัมพันธ์
- `/plan` - จับคู่จังหวะอารมณ์กับบทต่าง ๆ
- `/write` - เทคนิคบทสนทนาและเคมี
- `/analyze` - ตรวจสอบการปฏิบัติตามรูปแบบนิยายรัก

#### Mystery (สืบสวนสอบสวน)

**เงื่อนไขการเปิดใช้งาน**:
- พูดถึง "สืบสวน" "สอบสวน" "นักสืบ" "อาชญากรรม"
- เรื่องราวเกี่ยวข้องกับปริศนาและการสืบสวน

**สิ่งที่ให้**:
- หลักการเล่นที่ยุติธรรม
- กลยุทธ์การวางเบาะแส (3 ระดับ)
- การออกแบบเบาะแสหลอก
- การจัดการผู้ต้องสงสัย

**เวลาที่ประยุกต์ใช้**:
- `/specify` - กำหนดปริศนาหลัก
- `/plan` - วางแผนเส้นเวลาการวางเบาะแส
- `/write` - ตรวจสอบว่าเบาะแสเห็นได้แต่ไม่ชัดเจนเกินไป
- `/analyze` - ตรวจสอบการเล่นที่ยุติธรรม

#### Fantasy (แฟนตาซี)

**เงื่อนไขการเปิดใช้งาน**:
- พูดถึง "แฟนตาซี" "เวทมนตร์" "การสร้างโลก"
- เรื่องราวมีองค์ประกอบของโลกสมมติ

**สิ่งที่ให้**:
- กฎเวทมนตร์ของแซนเดอร์สัน
- กรอบการออกแบบระบบเวทมนตร์
- รายการตรวจสอบการสร้างโลก
- หลักการออกแบบเผ่าพันธุ์และสิ่งมีชีวิต

**เวลาที่ประยุกต์ใช้**:
- `/specify` - กำหนดองค์ประกอบโลกหลัก
- `/plan` - ตรวจสอบความสอดคล้องของกฎโลก
- `/write` - แสดงโลกผ่านการกระทำ
- `/analyze` - ตรวจสอบความสอดคล้องของกฎโลก

### 2. Writing Techniques Skills (เทคนิคการเขียน)

#### Dialogue Techniques (เทคนิคบทสนทนา)

**เงื่อนไขการเปิดใช้งาน**:
- กำลังเขียนฉากบทสนทนา
- ถามคำถามเกี่ยวกับบทสนทนาของตัวละคร

**สิ่งที่ให้**:
- การแสดงตัวละครผ่านภาษา
- การใช้คำซ่อนเร้นแทนการพูดตรง ๆ
- เทคนิคการพูดแทรกและการพูดซ้อน
- การหลีกเลี่ยงการบอกข้อมูลเกินจำเป็น

**เวลาที่ประยุกต์ใช้**:
- `/write` - ปรับปรุงฉากบทสนทนาโดยอัตโนมัติ
- แนะนำความสอดคล้องของน้ำเสียงตัวละครแบบเรียลไทม์
- ตรวจจับและเตือนบทสนทนาที่ตรงเกินไป

#### Scene Structure (โครงสร้างฉาก)

**เงื่อนไขการเปิดใช้งาน**:
- กำลังวางแผนหรือเขียนฉาก
- ถามคำถามเกี่ยวกับโครงสร้างบท

**สิ่งที่ให้**:
- โมเดลฉาก-ต่อเนื่อง
- กรอบเป้าหมาย-ความขัดแย้ง-หายนะ
- กลยุทธ์การจัดการความตึงเครียด
- การออกแบบจุดจังหวะ

**เวลาที่ประยุกต์ใช้**:
- `/plan` - ออกแบบลำดับฉาก
- `/write` - คำแนะนำการสร้างฉาก
- ให้เทมเพลตโครงร่างฉาก

### 3. Quality Assurance Skills (การรับประกันคุณภาพ)

#### Consistency Checker (ตัวตรวจสอบความสอดคล้อง)

**เงื่อนไขการเปิดใช้งาน**:
- ทำงานเบื้องหลัง **อัตโนมัติ** ระหว่างการเขียน
- ไม่ต้องเปิดใช้งานด้วยตนเอง

**สิ่งที่ตรวจสอบ**:
- ความสอดคล้องของตัวละคร (ลักษณะทางกายภาพ บุคลิก ความรู้)
- ความสอดคล้องของกฎโลก
- ความมีเหตุผลของเส้นเวลา

**รูปแบบการแจ้งเตือน**:

```
⚠️ การแจ้งเตือนการตรวจสอบความสอดคล้อง

ปัญหา: ลักษณะตัวละครไม่ตรงกัน
ตำแหน่ง: บทที่ 5 วรรคที่ 3
อ้างอิง: characters/mary.md บรรทัดที่ 15

ข้อความปัจจุบัน: "ดวงตาสีเขียวของแมรี่..."
ลักษณะที่กำหนดไว้: "สีตา: น้ำเงิน"

แนวทางแก้ไขที่เป็นไปได้:
1. เปลี่ยนข้อความปัจจุบันเป็น "ดวงตาสีน้ำเงิน"
2. อัปเดตแฟ้มข้อมูลตัวละคร (หากเปลี่ยนแปลงการตั้งค่า)
3. นี่เป็นคนละตัวละคร?

คุณต้องการให้ฉันแก้ไขอัตโนมัติหรือไม่?
```

**ระดับความรุนแรง**:
- 🔴 **สำคัญ**: แก้ไขทันที
- ⚠️ **คำเตือน**: แก้ไขโดยเร็ว
- 📝 **หมายเหตุ**: ควรพิจารณาตรวจสอบ

#### Workflow Guide (ตัวนำทางขั้นตอนการทำงาน)

**เงื่อนไขการเปิดใช้งาน**:
- ผู้ใช้พูดว่า "ฉันอยากเขียนนิยาย"
- เมื่อเริ่มโปรเจกต์ใหม่
- เมื่อเบี่ยงเบนจากขั้นตอนที่แนะนำ

**สิ่งที่ให้**:
- การนำทางระเบียบวิธีเจ็ดขั้นตอน
- คำอธิบายความสัมพันธ์การพึ่งพาของขั้นตอน
- คำแนะนำแนวทางปฏิบัติที่ดีที่สุด
- การจัดการปัญหาที่พบบ่อย

**การเตือนอย่างนุ่มนวล**:

```
ฉันสังเกตว่าคุณข้ามไปเขียนโดยตรง จุดแข็งของ Novel-writer 
คือการวางแผนอย่างเป็นระบบในระยะเริ่มต้น คุณต้องการรัน /constitution 
และ /specify เพื่อสร้างรากฐานที่แข็งแกร่งหรือไม่?
```

## การทำงานร่วมกันของ Skills และ Commands

### ตัวอย่างขั้นตอนการทำงานแบบเต็ม

```
ผู้ใช้: "ฉันจะเขียนนิยายรักแนวสืบสวน"

[เปิดใช้งานอัตโนมัติ]
✓ Romance Skill
✓ Mystery Skill  
✓ Workflow Guide Skill

AI: "เยี่ยมมาก! ใช้แนวทางที่เป็นระบบกันเถอะ เริ่มด้วยการรัน 
/constitution เพื่อกำหนดหลักการเขียน..."

ผู้ใช้รัน: /constitution
[Workflow Guide ให้คำแนะนำ]

ผู้ใช้รัน: /specify
[Romance + Mystery Skills ให้คำแนะนำเฉพาะประเภท]

ผู้ใช้รัน: /write
[Skills ที่เปิดใช้งานทั้งหมดทำงานร่วมกัน]
- Romance: จังหวะอารมณ์
- Mystery: การวางเบาะแส
- Dialogue Techniques: ปรับปรุงบทสนทนา
- Consistency Checker: ตรวจสอบเบื้องหลัง

[ตรวจพบปัญหา]
⚠️ การแจ้งเตือนความสอดคล้อง: ตัวละครในบทที่ 3 ไม่รู้ข้อมูล
แต่ในบทที่ 5 กลับรู้ทันที
```

## การกำหนดค่า Skills

### การปรับระดับความเข้มงวด

คุณสามารถปรับระดับความเข้มงวดของการตรวจสอบความสอดคล้อง:

```
"กรุณาใช้โหมดยืดหยุ่นในการตรวจสอบความสอดคล้อง 
เพราะนี่คือนิยายแฟนตาซีที่เวทมนตร์สามารถดัดแปลงความเป็นจริงได้"
```

**โหมด**:
- **โหมดเข้มงวด**: ทำเครื่องหมายความขัดแย้งทั้งหมด
- **โหมดยืดหยุ่น**: อนุญาตข้อยกเว้น "กฎเจ๋ง ๆ"
- **โหมดน้อยที่สุด**: ทำเครื่องหมายเฉพาะความขัดแย้งสำคัญ

### การปิดใช้งานการตรวจสอบเฉพาะ

หากความไม่สอดคล้องบางอย่างตั้งใจ:

```
"กรุณาปิดใช้งานการตรวจสอบเส้นเวลาสำหรับลำดับความฝัน - 
มันตั้งใจให้ไม่เป็นเส้นตรง"
```

## การทดสอบการเปิดใช้งาน Skills

อยากรู้ว่า Skill ไหนเปิดใช้งานอยู่? ถามตรง ๆ:

```
"Skills ใดที่เปิดใช้งานอยู่ในขณะนี้?"
```

AI จะบอกคุณว่ารายการ Skills ที่กำลังทำงานอยู่

## คำถามที่พบบ่อย

### Q: Skills จะเข้าไปยุ่งกับการสร้างสรรค์ของฉันมากเกินไปหรือไม่?

A: ไม่! หลักการออกแบบ Skills คือ:
- **แนะนำ ไม่บังคับ**: ให้คำแนะนำเชิงวิชาชีพ แต่คุณมีสิทธิ์ตัดสินใจขั้นสุดท้าย
- **เปิดใช้งานแบบรับฟัง**: ไม่รบกวนขั้นตอนการสร้างสรรค์ของคุณ
- **ปรับแต่งได้**: คุณสามารถปรับหรือปิดใช้งานได้

### Q: ฉันสามารถเปิดใช้งาน Skill ใดด้วยตนเองได้หรือไม่?

A: ไม่จำเป็น! แค่พูดถึงเนื้อหาที่เกี่ยวข้อง:
- อยากเปิดใช้งาน Romance Skill? พูดว่า "นี่คือนิยายรัก"
- อยากเปิดใช้งาน Mystery Skill? พูดว่า "นี่มีปริศนา"

### Q: Skills หลายตัวเปิดใช้งานพร้อมกันจะขัดแย้งกันหรือไม่?

A: ไม่! Skills ออกแบบมาให้ทำงานร่วมกัน:
- Romance + Mystery = รักสืบสวน
- Fantasy + Romance = แฟนตาซีรัก
- Skills ทั้งหมดรับรู้ถึงการมีอยู่ของกันและกัน

### Q: Skills จะเพิ่มเวลาตอบสนองหรือไม่?

A: น้อยมาก! Skills มีน้ำหนักเบา:
- ส่วนใหญ่ทำงานเบื้องหลัง
- ให้คำแนะนำเฉพาะเมื่อเกี่ยวข้อง
- ไม่กระทบความเร็วการสนทนาปกติ

### Q: ฉันจะรู้ได้อย่างไรว่าคำแนะนำมาจาก Skill ใด?

A: โดยปกติไม่จำเป็นต้องรู้! แต่ถ้าอยากรู้:
- คำแนะนำจะกล่าวถึงที่มา (เช่น "ตามรูปแบบนิยายรัก...")
- คุณสามารถถาม "คำแนะนำนี้มาจาก Skill ใด?"

## ความแตกต่างระหว่าง Skills และปลั๊กอิน

| คุณสมบัติ | Agent Skills | ปลั๊กอิน |
|------|-------------|------|
| **วิธีการเปิดใช้งาน** | AI ตัดสินใจอัตโนมัติ | ผู้ใช้ติดตั้ง |
| **ตำแหน่ง** | `.claude/skills/` | `plugins/` |
| **ประเภท** | ฐานความรู้ ระบบตรวจสอบ | คำสั่ง ผู้เชี่ยวชาญ ฟังก์ชัน |
| **ระดับการรับรู้** | ไม่รับรู้ | ชัดเจน (คำสั่งสแลช) |
| **วัตถุประสงค์** | ผู้ช่วยแบบรับฟัง | ฟังก์ชันที่เรียกใช้โดยตรง |

**พูดง่าย ๆ**:
- Skills = ความรู้เชิงวิชาชีพที่เปิดใช้งานอัตโนมัติ
- ปลั๊กอิน = เครื่องมือและฟังก์ชันที่เรียกใช้ด้วยตนเอง

Agent Skills คือ **ผู้ช่วยที่มองไม่เห็น** ของคุณ:
- ✨ เปิดใช้งานอัตโนมัติ ไม่ต้องเรียกใช้
- 🎯 ประยุกต์ใช้ความรู้เชิงวิชาชีพอย่างแม่นยำ
- 🔍 ตรวจพบปัญหาเชิงรุก
- 🤝 ทำงานร่วมกับคำสั่ง
- 🎛️ ปรับแต่งและกำหนดค่าได้

**คุณมุ่งเน้นการสร้างสรรค์ Skills รับผิดชอบคุณภาพ!**
