# การตรวจสอบตนเองและปรับแก้เพื่อเพิ่มความเป็นมนุษย์

เป้าหมาย: ช่วยให้ผู้เขียนค้นพบร่องรอยของรูปแบบสำเร็จรูป คำพูดที่ไร้สาระ และปัญหาจังหวะที่ซ้ำซากในข้อความ พร้อมให้คำแนะนำในการปรับแต่งระดับประโยค/ย่อหน้า เน้น “ให้ดูเหมือนเป็นคุณเขียนเอง” มากกว่าการหลีกเลี่ยงการตรวจจับใดๆ

## วิธีการใช้

```
/authenticity-audit

[วางข้อความที่ต้องการตรวจสอบ]
```

## มิติการตรวจสอบ (ให้จำนวนครั้งและตัวอย่าง)

หากมีไฟล์ `spec/knowledge/audit-config.json` ให้ใช้คำเชื่อม รายการคำไร้สาระ และเกณฑ์ความยาวประโยคตามนั้น มิฉะนั้นให้ใช้กฎเริ่มต้นในตัว

- ความหนาแน่นของคำเชื่อมแบบสำเร็จรูป: เช่น “ประการแรก ประการต่อมา นอกจากนี้ โดยสรุป” ฯลฯ – นับจำนวนและให้คำแนะนำในการแทนที่
- สำนวนที่ไร้สาระ: เช่น “ในระดับหนึ่ง เป็นที่ทราบกันดี ในหลายวงการ” – ให้ข้อเสนอในการปรับแก้ให้มีความเฉพาะเจาะจงมากขึ้น
- ความแปรปรวนของความยาวประโยค: คำนวณความยาวประโยคเฉลี่ยและความแปรปรวน ทำเครื่องหมายช่วง “ประโยคยาวติดกัน” หรือ “ประโยคสั้นติดกัน” พร้อมคำแนะนำในการแบ่ง/รวมเพื่อปรับจังหวะ
- สำนวนซ้ำซากจำเจ: ระบุวลีที่ใช้กันทั่วไปและเสนอทางเลือกสองถึงสามแบบที่เหมาะสมกับเนื้อหามากกว่า
- การใช้คำนามธรรมมากเกินไป: ชี้ให้เห็นย่อหน้าที่มีความหนาแน่นของคำนามธรรมสูง และแนะนำให้เพิ่มรายละเอียดที่สัมผัสได้
- ความสอดคล้องกับบุคลิก: เปรียบเทียบกับคำติดปาก/คำศัพท์ใน `.specify/memory/personal-voice.md` ชี้ให้เห็นการขาด “สัญญาณเฉพาะตัว” และแนะนำจุดที่สามารถสอดแทรกได้อย่างเป็นธรรมชาติ

## ผลลัพธ์การปรับแก้

- แบ่งเป็นช่วง: ประโยคต้นฉบับ → คำอธิบายปัญหา → เวอร์ชันปรับเล็กน้อย → เวอร์ชันที่เป็นส่วนตัวมากขึ้น
- ควบคุมขอบเขต: ปรับเล็กน้อยก่อน รักษาความหมายเดิมไว้ให้มากที่สุด ปรับมากกว่าเดิมเฉพาะเมื่อจำเป็น
- รักษารูปแบบ: ไม่เปลี่ยนแปลงรูปแบบเอกสารและมุมมองการเล่าเรื่อง เว้นแต่ผู้ใช้จะขอให้เปลี่ยนโดยชัดเจน

## บัตรคำแนะนำท้ายสุด

- ให้คำแนะนำเฉพาะบุคคล 5–8 ข้อที่ “สามารถใช้ได้ทันทีในการเขียนครั้งถัดไป” โดยอิงจากปัญหาที่พบในข้อความนี้
