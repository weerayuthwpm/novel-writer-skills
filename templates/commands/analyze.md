ทำการวิเคราะห์แบบครบวงจรอัจฉริยะสำหรับโปรเจกต์นิยาย โดยเลือกดำเนินการ **การวิเคราะห์ความสอดคล้องของเฟรมเวิร์ก** (ก่อน write) หรือ **การวิเคราะห์คุณภาพเนื้อหา** (หลัง write) โดยอัตโนมัติตามขั้นตอนการสร้างสรรค์งานในปัจจุบัน
---

## แนวคิดหลัก

**หนึ่งคำสั่ง สองความอัจฉริยะ**:
- 📐 **การวิเคราะห์เฟรมเวิร์ก**: ตรวจสอบความสอดคล้องของข้อกำหนด แผน และภารกิจก่อนเขียน (คล้าย spec-kit)
- 📝 **การวิเคราะห์เนื้อหา**: ตรวจสอบคุณภาพและความสอดคล้องของเนื้อหาที่เขียนเสร็จแล้วหลังเขียน

**เรียบง่ายแต่ไม่ขาดความละเอียด**:
- ผู้ใช้เพียงแค่รัน `/analyze` ระบบจะตัดสินใจโดยอัตโนมัติว่าควรดำเนินการวิเคราะห์แบบใด
- รองรับการระบุโหมดด้วยตนเอง: `$ARGUMENTS --type=framework` หรือ `--type=content`

## ขั้นตอนการดำเนินงาน

### 1. การตรวจจับขั้นตอนอัจฉริยะ

รัน `{SCRIPT}` เพื่อรับสถานะการสร้างสรรค์งานปัจจุบัน:

```json
{
  "analyze_type": "framework|content",
  "chapter_count": 0,
  "has_spec": true,
  "has_plan": true,
  "has_tasks": true,
  "story_dir": "/path/to/story",
  "reason": "คำอธิบายเหตุผล"
}
```

### 2. ตรรกะการตัดสินใจ

วิเคราะห์พารามิเตอร์ผู้ใช้ `$ARGUMENTS`:

**โหมดระบุด้วยตนเอง** (มีความสำคัญสูงสุด):
- มี `--type=framework` → บังคับวิเคราะห์เฟรมเวิร์ก
- มี `--type=content` → บังคับวิเคราะห์เนื้อหา

**🆕 โหมดวิเคราะห์เฉพาะด้าน** (เพิ่มใหม่):
- มี `--focus=opening` → วิเคราะห์เฉพาะด้านบทเปิด (เน้นวิเคราะห์ 3 บทแรก)
- มี `--focus=pacing` → วิเคราะห์เฉพาะด้านจังหวะ (เน้นวิเคราะห์การกระจายของจุดพีค/ความขัดแย้ง)
- มี `--focus=character` → วิเคราะห์เฉพาะด้านตัวละคร (เน้นวิเคราะห์ส่วนโค้งของตัวละคร)
- มี `--focus=foreshadow` → วิเคราะห์เฉพาะด้านการปูทาง (เน้นวิเคราะห์การวางและเก็บเกี่ยวปม)
- มี `--focus=logic` → วิเคราะห์เฉพาะด้านตรรกะ (เน้นค้นหาช่องโหว่ทางตรรกะ)
- มี `--focus=style` → วิเคราะห์เฉพาะด้านสไตล์ (เน้นตรวจสอบความสม่ำเสมอของสำนวนการเขียน)

**โหมดตัดสินใจอัตโนมัติ**:
- จำนวนบท = 0 → **วิเคราะห์เฟรมเวิร์ก**
- จำนวนบท < 3 → **วิเคราะห์เฟรมเวิร์ก** (แต่แนะนำให้เขียนต่อได้)
- จำนวนบท ≥ 3 → **วิเคราะห์เนื้อหา**

### 3. ดำเนินการวิเคราะห์ตามที่เลือก

ตามผลการตัดสินใจ ให้ดำเนินการวิเคราะห์หนึ่งในสองแบบดังต่อไปนี้

## โหมด A: การวิเคราะห์ความสอดคล้องของเฟรมเวิร์ก

**เป้าหมาย**: ตรวจสอบก่อนเขียนว่าการเตรียมงานเพียงพอหรือไม่ และรับประกันว่าไม่มีความขัดแย้งระหว่างข้อกำหนด แผน และภารกิจ

### A1. โหลดเอกสารอ้างอิง

- เอกสารรัฐธรรมนูญ: `.specify/memory/constitution.md`
- เอกสารข้อกำหนด: `stories/*/specification.md`
- เอกสารแผน: `stories/*/creative-plan.md`
- เอกสารภารกิจ: `stories/*/tasks.md`

### A2. การวิเคราะห์ความครอบคลุม

ตรวจสอบว่าความต้องการทั้งหมดในข้อกำหนดมีแผนและภารกิจที่สอดคล้องหรือไม่:

```markdown
## รายงานการวิเคราะห์ความครอบคลุม

### ความครอบคลุมความต้องการ P0
- [ความต้องการ 1: เส้นทางการเติบโตของตัวละครหลัก] → ✅ แผนบทที่ 3 ภารกิจ #5-8
- [ความต้องการ 2: การตั้งค่าตัวร้าย] → ⚠️ กล่าวถึงในแผน แต่ไม่มีภารกิจเฉพาะ
- [ความต้องการ 3: การวางปมปริศนา] → ❌ ไม่ครอบคลุมทั้งในแผนและภารกิจ

### ความครอบคลุมความต้องการ P1
ความครอบคลุม: 75% (3/4)

### ความครอบคลุมความต้องการ P2
ความครอบคลุม: 50% (2/4)

### ความสมบูรณ์ของภารกิจ
- ทุกบทที่วางแผนไว้มีภารกิจที่สอดคล้องหรือไม่: ⚠️ บทที่ 10-12 ขาดภารกิจ
- ภารกิจครอบคลุมฉากสำคัญทั้งหมดหรือไม่: ✅ ใช่
```

### A3. การตรวจสอบความสอดคล้อง

ตรวจสอบว่ามีความขัดแย้งระหว่างเอกสารหรือไม่:

```markdown
## รายงานการตรวจสอบความสอดคล้อง

### ข้อกำหนด ↔ แผน
- ✅ การแสดงธีมสอดคล้องกัน
- ⚠️ ข้อกำหนดต้องการ "จังหวะรวดเร็ว" แต่แผน 5 บทแรกมีจังหวะค่อนข้างช้า
- ❌ ข้อกำหนดห้าม "ฉากความรักมากเกินไป" แต่แผนบทที่ 6-8 มีเส้นเรื่องความรักจำนวนมาก

### แผน ↔ ภารกิจ
- ✅ ทุกบทในแผนมีภารกิจ
- ⚠️ จำนวนคำรวมโดยประมาณของภารกิจ 150K คำ แต่เป้าหมายแผนคือ 100K คำ
- ❌ แผนกำหนดให้บทที่ 5 เป็นจุดไคลแมกซ์ แต่ภารกิจระบุว่าเป็น "บทเปลี่ยนผ่าน"

### การปฏิบัติตามรัฐธรรมนูญ
- ✅ แผนสอดคล้องกับค่านิยมของรัฐธรรมนูญการสร้างสรรค์
- ✅ การแบ่งภารกิจสอดคล้องกับมาตรฐานคุณภาพ
```

### A4. การเตือนปัญหาตรรกะล่วงหน้า

วิเคราะห์ช่องโหว่ทางตรรกะที่อาจเกิดขึ้นในการออกแบบโครงเรื่อง:

```markdown
## การเตือนปัญหาตรรกะล่วงหน้า

### ความขัดแย้งของเส้นเวลา
- ⚠️ บทที่ 3 คือ "สามปีต่อมา" แต่บทที่ 5 ตัวละครพูดถึง "เรื่องเมื่อสองปีก่อน" เวลาไม่ตรงกัน

### ความขัดแย้งของความสามารถตัวละคร
- ❌ บทที่ 2 ตัวละครหลัก "ไม่รู้จักศิลปะการต่อสู้" บทที่ 4 คำอธิบายภารกิจ "ใช้ดาบปราบศัตรู"

### ปมที่ยังไม่ได้วางแผน
- ⚠️ บทที่ 1 วางปม "แผ่นป้ายลึกลับ" แต่บทต่อมาไม่มีแผนเก็บเกี่ยว
```

### A5. การประเมินความพร้อม

ประเมินว่าสามารถเริ่มเขียนได้หรือไม่:

```markdown
## การประเมินความพร้อม

### เงื่อนไขจำเป็น (P0)
- [x] ข้อกำหนดครบถ้วนและชัดเจน
- [x] แผนครอบคลุมความต้องการ P0 ทั้งหมด
- [ ] การแบ่งภารกิจสมบูรณ์ (ขาดภารกิจ 3 บท)
- [ ] ไม่มีข้อขัดแย้งทางตรรกะร้ายแรง (พบ 2 จุด)

### เงื่อนไขแนะนำ (P1)
- [x] แฟ้มตัวละครสมบูรณ์
- [ ] เอกสารการตั้งค่าโลกยังไม่ละเอียดเพียงพอ
- [x] การวางแผนเส้นเวลาชัดเจน

### คะแนนรวม: 6/10

**ข้อเสนอแนะ**:
1. 🔴 ต้องแก้ไข: เพิ่มภารกิจบทที่ 10-12
2. 🔴 ต้องแก้ไข: แก้ไขข้อขัดแย้งของเส้นเวลาและความสามารถตัวละคร
3. 🟡 แนะนำให้ปรับปรุง: เพิ่มเอกสารการตั้งค่าโลก
4. 🟢 เลือกได้: ปรับการออกแบบจังหวะ 5 บทแรก

**สรุป**: ปัจจุบัน **ไม่แนะนำให้เริ่มเขียน** กรุณาแก้ไขปัญหา P0 ก่อน
```

### B1. โหลดเกณฑ์อ้างอิง

- เอกสารรัฐธรรมนูญ: `.specify/memory/constitution.md`

### B2. การตรวจสอบการปฏิบัติตามรัฐธรรมนูญ

ตรวจสอบว่างานเป็นไปตามหลักการของรัฐธรรมนูญการสร้างสรรค์หรือไม่:

```markdown
## รายงานการปฏิบัติตามรัฐธรรมนูญ

### การตรวจสอบค่านิยมหลัก
- [x] หลักการค่านิยม 1: ธีมเชิงบวกและสร้างสรรค์ ✅
- [x] หลักการค่านิยม 2: หลีกเลี่ยงเนื้อหาที่ไม่เหมาะสม ✅
- [ ] หลักการค่านิยม 3: เคารพประเพณีทางวัฒนธรรม ⚠️ บทที่ 7 มีคำอธิบายที่น่าสงสัย

### การตรวจสอบมาตรฐานคุณภาพ
- ความสอดคล้องทางตรรกะ: 8/10 ⚠️ บทที่ 3 และบทที่ 6 มีความขัดแย้งเล็กน้อย
- ความลึกของตัวละคร: 7/10 (ตัวละครหลักมีมิติหลากหลาย ตัวละครรองค่อนข้างบาง)
- ระดับภาษา: 8/10 (ความลื่นไหลดี แต่บางส่วนสามารถเสริมการพรรณนาได้)

### ความสม่ำเสมอของสไตล์
- รูปแบบการเล่าเรื่อง: สม่ำเสมอ ✅
- รูปแบบภาษา: สม่ำเสมอ ✅
- การควบคุมจังหวะ: ช่วงต้นช้า ช่วงปลายเร็ว โดยรวมสมเหตุสมผล ✅

**คะแนนรวม: 8/10**
```

### B3. การวิเคราะห์ความสอดคล้องกับข้อกำหนด

ตรวจสอบว่าผลลัพธ์เป็นไปตามข้อกำหนดหรือไม่:

```markdown
## การวิเคราะห์ความสอดคล้องกับข้อกำหนด

### ความครอบคลุมความต้องการหลัก
#### P0 (ต้องมี)
- [ความต้องการ 1: ความขัดแย้งระหว่างพ่อลูก] → ✅ บทที่ 2-4 แสดงให้เห็นอย่างเต็มที่
- [ความต้องการ 2: การวางปมปริศนา] → ⚠️ บทที่ 5 มีปมไม่เพียงพอ
- [ความต้องการ 3: ตัวร้ายมีมิติ] → ❌ ตัวร้ายยังไม่ปรากฏตัวอย่างเป็นทางการ

ความครอบคลุม: 67% (2/3)

#### P1 (ควรมี)
ความครอบคลุม: 75% (3/4)

#### P2 (อาจมี)
ความครอบคลุม: 50% (2/4)

### ระดับการบรรลุเป้าหมาย
- ความเหมาะสมกับกลุ่มเป้าหมาย: 85% (จังหวะและเนื้อเรื่องสอดคล้องกับความชอบของกลุ่มเป้าหมาย)
- ความสอดคล้องกับตำแหน่งทางการตลาด: 80% (จุดขายที่แตกต่างชัดเจน แต่ยังต้องเสริม)
- การบรรลุเกณฑ์ความสำเร็จ: 5/8 ⚠️ ตัวชี้วัดบางส่วนยังไม่บรรลุ

### การปฏิบัติตามข้อจำกัด
- เส้นสีแดงของเนื้อหา: ✅ ไม่มีการละเมิด
- ข้อจำกัดการสร้างสรรค์: ✅ จำนวนคำ ความถี่ในการอัปเดตสอดคล้องตามข้อกำหนด
- ข้อจำกัดทางเทคนิค: ✅ เป็นไปตามข้อกำหนดรูปแบบแพลตฟอร์ม

**คะแนนรวม: 7/10**
```

### B4. การวิเคราะห์การดำเนินการตามแผน

ประเมินความเบี่ยงเบนระหว่างการดำเนินการจริงกับแผน:

```markdown
## การวิเคราะห์การดำเนินการตามแผน

### การเปรียบเทียบโครงสร้างบท

|------|------|--------------------------|

| บทที่ 2: ขยายความขัดแย้ง | ✅ เสร็จ | ปรับเล็กน้อย เพิ่มการปูทาง |
| บทที่ 3: จุดเปลี่ยน | ⚠️ เสร็จ | จุดเปลี่ยนเกิดขึ้นก่อนที่ตอนจบบทที่ 2 |

| บทที่ 5: บทนำสู่ไคลแมกซ์ | ❌ เลื่อน | กลายเป็นบทเปลี่ยนผ่านจริง |

### เส้นทางการพัฒนาตัวละคร
```

### B7. สร้างข้อเสนอแนะปรับปรุง

จากผลการวิเคราะห์ ให้ข้อเสนอแนะเฉพาะ:

```markdown
## ข้อเสนอแนะปรับปรุง

### การแก้ไขด่วน (P0)
1. **ความขัดแย้งของเส้นเวลา**
   - ผลกระทบ: ทำลายความเชื่อมั่นของผู้อ่าน ส่งผลต่อความรัดกุมทางตรรกะ
   - คำแนะนำ: ปรับการระบุเวลาในบทที่ 3 และบทที่ 6 ให้เป็น "สองปีต่อมา" ให้สอดคล้องกัน
   - ตำแหน่ง: บทที่ 3 ตอนที่ 2, บทที่ 6 ตอนที่ 4

2. **ความขัดแย้งของความสามารถตัวละคร**
   - ผลกระทบ: ส่งผลต่อความน่าเชื่อถือของตัวละครอย่างรุนแรง
   - คำแนะนำ: เพิ่มเนื้อเรื่องเปลี่ยนผ่าน "การเรียนรู้ศิลปะการต่อสู้" ระหว่างบทที่ 4-5 หรือลบการพรรณนาศิลปะการต่อสู้ในบทที่ 6
   - ตำแหน่ง: บทที่ 2 ตอนที่ 5, บทที่ 6 ตอนที่ 3

### ข้อเสนอแนะปรับปรุง (P1)
1. **ปมในบทที่ 5 ไม่เพียงพอ**
   - ปัจจุบัน: ท้ายบทที่ 5 ราบเรียบ ขาดเบ็ด
   - คำแนะนำ: เพิ่มเหตุการณ์ไม่คาดฝันหรือข้อมูลที่ท้ายบท สร้างความคาดหวังให้ผู้อ่าน
   - ผลที่คาดหวัง: เพิ่มอัตราการคงอยู่ของผู้อ่าน

2. **หน้าที่ตัวละครรอง B ยังไม่ปรากฏ**
   - ปัจจุบัน: ตัวละครรอง B ปรากฏแต่บทบาทไม่ชัดเจน
   - คำแนะนำ: จัดให้ตัวละครรอง B มีบทบาทสำคัญในบทที่ 8-9 เชื่อมโยงกับการปูทางตอนต้น
   - ผลที่คาดหวัง: เพิ่มความมีตัวตนของตัวละครรอง เพิ่มมิติของเรื่อง

### การปรับปรุงระยะยาว (P2)
1. **การเปิดเผยการตั้งค่าโลกเร็วเกินไป**
   - เหตุผล: อาจส่งผลต่อการสร้างบรรยากาศลึกลับในระยะหลัง
   - แนวทาง: ประเมินว่าต้องปรับจังหวะการเปิดเผยภายหลัง หรือเพิ่มการตั้งค่าระดับลึกขึ้น
   - จังหวะเวลา: ตัดสินใจก่อนบทที่ 10

**การจัดลำดับความสำคัญ**: P0-1 (เส้นเวลา) → P0-2 (ความสามารถ) → P1-1 (ปม) → P1-2 (ตัวละครรอง)
```

```text
4-5 คะแนน: พอใช้ มีปัญหามาก
1-3 คะแนน: แย่ ต้องเขียนใหม่
```

## ข้อควรระวัง

### ฉลาดแต่ควบคุมได้
- โหมดอัตโนมัติครอบคลุม 90% ของสถานการณ์การใช้งาน
- โหมดด้วยตนเองรองรับความต้องการพิเศษ
- ผู้ใช้ไม่ต้องจำกฎที่ซับซ้อน

### 🆕 สถานการณ์การใช้การวิเคราะห์เฉพาะด้าน

**เมื่อใดควรใช้การวิเคราะห์เฉพาะด้าน?**

1. **--focus=opening**: ใช้ทันทีหลังจากเสร็จ 3 บทแรก
   - บทเปิดเป็นกุญแจสำคัญในการรักษาผู้อ่าน
   - กฎทองบทเปิดมีข้อกำหนดที่เข้มงวด
   - การค้นหาปัญหาแต่เนิ่น ๆ มีต้นทุนต่ำกว่า

2. **--focus=pacing**: ใช้ทุก 10-15 บท
   - ตรวจสอบว่าจังหวะเป็นไปตามคาดหรือไม่
   - ประเมินความสมเหตุสมผลของการกระจายจุดพีค/ความขัดแย้ง
   - ปรับจังหวะตาม rhythm-config

3. **--focus=character**: ใช้หลังจากจุดเปลี่ยนสำคัญ
   - หลังจากตัวละครหลักเผชิญเหตุการณ์สำคัญ
   - เมื่อตัวละครรองปรากฏหรือออกไป
   - เมื่อความสัมพันธ์ของตัวละครเปลี่ยนแปลง

4. **--focus=foreshadow**: ใช้หลังจากแต่ละเล่มเสร็จ
   - ตรวจสอบว่าปมขาดหายหรือไม่
   - ประเมินคุณภาพการวางปม
   - วางแผนจังหวะการเก็บเกี่ยวล่วงหน้า

5. **--focus=logic**: ใช้หลังจากปรับโครงเรื่อง
   - หลังจากแก้ไขการตั้งค่าสำคัญ
   - หลังจากปรับเส้นเวลา
   - หลังจากเพิ่ม/ลดเนื้อหาบท

6. **--focus=style**: ใช้ก่อนแก้ไขเป็นชุด
   - เปรียบเทียบ style-reference ตรวจสอบความสม่ำเสมอ
   - ค้นหาสำนวน AI และคำต้องห้าม
   - รับประกันสไตล์สอดคล้องกับงานอ้างอิง

**ความสัมพันธ์ระหว่างการวิเคราะห์เฉพาะด้านและการวิเคราะห์全面**:
- **การวิเคราะห์全面** (ค่าเริ่มต้น): เหมาะสำหรับการตรวจสอบเป็นระยะ (ทุก 5-10 บท)
- **การวิเคราะห์เฉพาะด้าน** (--focus): เหมาะสำหรับการปรับปรุงแบบเจาะจง (เมื่อพบปัญหา)

**ขั้นตอนการทำงานที่แนะนำ**:
1. ทุก ๆ 5-10 บทที่เขียนเสร็จ → `/analyze` (全面)
2. พบปัญหาบทเปิด → `/analyze --focus=opening`
3. รู้สึกว่าจังหวะไม่ถูกต้อง → `/analyze --focus=pacing`
4. ไม่แน่ใจตรรกะ → `/analyze --focus=logic`
5. ตรวจสอบก่อนแก้ไข → `/analyze --focus=style`

### เป็นกลางและสร้างสรรค์
- วิเคราะห์ตามข้อมูลและมาตรฐาน

- ให้ข้อเสนอแนะที่นำไปปฏิบัติได้จริง

- ติดตามประสิทธิผลของการปรับปรุง
