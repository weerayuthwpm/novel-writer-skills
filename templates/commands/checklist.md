# รายการตรวจสอบคุณภาพ (Checklist)

สร้างหรือดำเนินการตรวจสอบรายการคุณภาพ รองรับสองโหมด:

## 🎯 ประเภทการตรวจสอบที่รองรับ

### ประเภทที่หนึ่ง: การตรวจสอบคุณภาพข้อกำหนด (แบบสร้างคำถาม)
ตรวจสอบคุณภาพของเอกสารการวางแผนเอง (คล้าย "Unit Test ของความต้องการ"):

- `คุณภาพโครงเรื่อง` - ตรวจสอบ outline.md ด้านความสมบูรณ์ ความชัดเจน ความสอดคล้อง
- `การตั้งค่าตัวละคร` - ตรวจสอบ spec/knowledge/characters.md
- `โลกทัศน์` - ตรวจสอบ spec/knowledge/world-setting.md และเอกสารที่เกี่ยวข้อง
- `แผนการสร้างสรรค์` - ตรวจสอบ creative-plan.md / specification.md
- `การจัดการปม` - ตรวจสอบคำจำกัดความของปมใน spec/tracking/plot-tracker.json

### ประเภทที่สอง: การตรวจสอบเนื้อหา (แบบรายงานผลลัพธ์)
สแกนบทที่เขียนแล้ว ตรวจสอบเนื้อหาจริง:

- `ความสอดคล้องของโลกทัศน์` - สแกนเนื้อหาบท ตรวจสอบคำอธิบายโลกทัศน์ที่ขัดแย้ง
- `การจัดแนวเนื้อเรื่อง` - เปรียบเทียบความคืบหน้ากับโครงเรื่อง ตรวจสอบการพัฒนาเนื้อเรื่อง
- `การซิงโครไนซ์ข้อมูล` - ตรวจสอบความสอดคล้องของไฟล์ JSON ที่ใช้ติดตามทั้งหมด
- `เส้นเวลา` - ตรวจสอบความต่อเนื่องทางตรรกะของเหตุการณ์ตามเวลา
- `สถานะการเขียน` - ตรวจสอบความพร้อมในการเขียนและสถานะภารกิจ

## ขั้นตอนการดำเนินงาน

### 1. ระบุประเภทการตรวจสอบ

ตามอินพุตของผู้ใช้ ให้กำหนดประเภทการตรวจสอบ (คุณภาพข้อกำหนด vs การตรวจสอบเนื้อหา):

**การจับคู่คำสำคัญ**:
- "โครงเรื่อง", "คุณภาพ" → ประเภทคุณภาพข้อกำหนด: คุณภาพโครงเรื่อง
- "ตัวละคร", "การตั้งค่า" → ประเภทคุณภาพข้อกำหนด: การตั้งค่าตัวละคร
- "โลกทัศน์" + "คุณภาพ/ความสมบูรณ์/ข้อกำหนด" → ประเภทคุณภาพข้อกำหนด: โลกทัศน์
- "โลกทัศน์" + "ความสอดคล้อง/ตรวจสอบ/สแกน" → ประเภทการตรวจสอบเนื้อหา: ความสอดคล้องของโลกทัศน์
- "แผนการสร้างสรรค์", "การวางแผน" → ประเภทคุณภาพข้อกำหนด: แผนการสร้างสรรค์
- "ปม" → ประเภทคุณภาพข้อกำหนด: การจัดการปม
- "เนื้อเรื่อง", "การจัดแนว", "ความคืบหน้า" → ประเภทการตรวจสอบเนื้อหา: การจัดแนวเนื้อเรื่อง
- "ข้อมูล", "ซิงค์", "ความสอดคล้อง" → ประเภทการตรวจสอบเนื้อหา: การซิงโครไนซ์ข้อมูล
- "เส้นเวลา", "เวลา" → ประเภทการตรวจสอบเนื้อหา: เส้นเวลา
- "สถานะการเขียน", "ความพร้อม" → ประเภทการตรวจสอบเนื้อหา: สถานะการเขียน

หากอินพุตของผู้ใช้ไม่ชัดเจน ให้ถามเพื่อเลือก

### 2. ดำเนินการตรรกะการตรวจสอบที่เกี่ยวข้อง

#### การตรวจสอบประเภทคุณภาพข้อกำหนด (สร้าง Checklist แบบคำถาม)

ดำเนินการตรรกะการตรวจสอบคุณภาพความต้องการแบบ spec-kit:

##### 2.1 การตรวจสอบคุณภาพโครงเรื่อง

**เป้าหมาย**: ตรวจสอบว่า outline.md มีความสมบูรณ์ ความชัดเจน และความสอดคล้องที่ดีหรือไม่

**อ่านไฟล์**:
- `outline.md` หรือ `stories/*/outline.md`
- `spec/tracking/plot-tracker.json` (ถ้ามี)

**มิติการสร้างรายการตรวจสอบ**:

**ความสมบูรณ์ (Completeness)**:
- มีการกำหนดเงื่อนไขการ触发และผลลัพธ์สำหรับโหนดเนื้อเรื่องหลักทุกโหนดหรือไม่?
- มีการระบุเป้าหมายของเนื้อเรื่องในแต่ละเล่ม/บทอย่างชัดเจนหรือไม่?
- ครอบคลุมส่วนโค้งการเติบโตของตัวละครหลักทั้งหมดหรือไม่?
- มีการกำหนดเส้นทางของการขยายความขัดแย้งหลักหรือไม่?
- มีการระบุไคลแมกซ์และตอนจบของเรื่องอย่างชัดเจนหรือไม่?

**ความชัดเจน (Clarity)**:
- เงื่อนไขการ触发ของโหนดเนื้อเรื่องสามารถตรวจสอบได้อย่างเป็นรูปธรรมหรือไม่?
- การจัดสรรบทมีพื้นฐานที่ชัดเจน (เช่น จำนวนคำ ความหนาแน่นของเนื้อเรื่อง) หรือไม่?
- แรงจูงใจของตัวละครถูกวัดด้วยเหตุการณ์ที่เป็นรูปธรรมหรือไม่?
- คำอธิบายฉากหลีกเลี่ยงคำที่คลุมเครือ ("ที่ไหนสักแห่ง" "ช่วงเวลาหนึ่ง") หรือไม่?

**ความสอดคล้อง (Consistency)**:
- เส้นเรื่องมีความขัดแย้งกันก่อนหลังหรือไม่?
- พฤติกรรมตัวละครสอดคล้องกับการตั้งค่าหรือไม่?
- ช่วงเวลามีความสมเหตุสมผลหรือไม่?
- กฎของโลกทัศน์สอดคล้องกับคำอธิบายในโครงเรื่องหรือไม่?

**ความสามารถในการวัด (Measurability)**:
- การจัดสรรบทมีความเป็นไปได้สมเหตุสมผลหรือไม่ (เช่น 2,000-4,000 คำต่อบท)?
- ช่วงเวลาเก็บเกี่ยวปมมีการระบุอย่างชัดเจน (หมายเลขบทหรือช่วงบท) หรือไม่?
- การเติบโตของตัวละครมีเหตุการณ์สำคัญที่ชัดเจนหรือไม่?

**ความครอบคลุม (Coverage)**:
- พิจารณาประเภทฉากหลักทั้งหมด (ความขัดแย้ง ชีวิตประจำวัน จุดเปลี่ยน) หรือไม่?
- ครอบคลุมบทบาทของตัวละครหลักทั้งหมดหรือไม่?
- รวมการวางและเก็บเกี่ยวปมที่จำเป็นหรือไม่?

##### 2.2 การตรวจสอบการตั้งค่าตัวละคร

**อ่านไฟล์**:

- `spec/tracking/character-state.json`

**มิติการสร้างรายการตรวจสอบ**:

**ความสมบูรณ์**:
- ตัวละครหลักมีข้อมูลพื้นฐาน (ชื่อ อายุ บทบาท รูปลักษณ์) หรือไม่?
- มีการกำหนดแรงจูงใจหลักและเป้าหมายของตัวละครหรือไม่?
- มีการกำหนดลักษณะนิสัยและรูปแบบพฤติกรรมของตัวละครหรือไม่?
- มีการกำหนดภูมิหลังของตัวละครหรือไม่?
- มีการกำหนดความสามารถและข้อจำกัดของตัวละครหรือไม่?

**ความชัดเจน**:
- แรงจูงใจของตัวละครสามารถตรวจสอบได้อย่างเป็นรูปธรรมหรือไม่ (ไม่ใช่ "ต้องการประสบความสำเร็จ" แต่เป็น "ต้องการเปลี่ยนแปลงชะตากรรมของครอบครัวผ่านการสอบราชการ")?
- ลักษณะนิสัยสะท้อนผ่านพฤติกรรมที่เป็นรูปธรรมหรือไม่?
- เป้าหมายของตัวละครสามารถวัดหรือมีเกณฑ์การบรรลุที่ชัดเจนหรือไม่?

**ความสอดคล้อง**:
- การตั้งค่าตัวละครสอดคล้องกับพฤติกรรมในโครงเรื่องหรือไม่?
- คำอธิบายตัวละครในเอกสารต่าง ๆ สอดคล้องกันหรือไม่?
- ความสัมพันธ์ของตัวละครมีการกำหนดแบบสมมาตรหรือไม่ (ความสัมพันธ์ของ A ต่อ B เทียบกับ B ต่อ A)?

**ความสามารถในการวัด**:
- การเติบโตของตัวละครมีระยะที่ชัดเจนหรือไม่?
- การเปลี่ยนแปลงความสามารถของตัวละครสามารถติดตามได้หรือไม่?

- คำศัพท์เฉพาะมีคำจำกัดความที่ชัดเจนหรือไม่?

**ความสอดคล้อง**:
- การตั้งค่าโลกทัศน์ในเอกสารต่าง ๆ สอดคล้องกันหรือไม่?
- กฎของโลกทัศน์มีความขัดแย้งภายในหรือไม่?
- สอดคล้องกับคำอธิบายในโครงเรื่องหรือไม่?

**ความครอบคลุม**:
- ครอบคลุมสถานที่ทั้งหมดที่เกี่ยวข้องในเรื่องหรือไม่?
- มีการกำหนดกฎหรือความสามารถพิเศษทั้งหมดที่ปรากฏหรือไม่?

**มิติการสร้างรายการตรวจสอบ**:

**ความสมบูรณ์**:
- มีการบันทึกปมที่วางแผนไว้ทั้งหมดหรือไม่?
- แต่ละปมมีการกำหนดบทที่วางและบทที่เก็บเกี่ยวหรือไม่?
- มีการกำหนดประเภทและความสำคัญของปมหรือไม่?

**ความชัดเจน**:
- คำอธิบายเนื้อหาปมชัดเจนหรือไม่?
- วิธีการเก็บเกี่ยวมีความชัดเจนหรือไม่?

**ความสามารถในการวัด**:
- ช่วงเวลาเก็บเกี่ยวมีหมายเลขบทหรือช่วงบทที่ชัดเจนหรือไม่?
- มีการกำหนดความหนาแน่นของการวางปม (หลีกเลี่ยงปมที่วางแล้วยังไม่ได้เก็บเกี่ยวมากเกินไป) หรือไม่?

**ความสอดคล้อง**:
- ปมสอดคล้องกับเนื้อเรื่องในโครงเรื่องหรือไม่?
- ฟิลด์ planted และ resolved สอดคล้องกันหรือไม่?

#### การตรวจสอบประเภทการตรวจสอบเนื้อหา (เรียกใช้สคริปต์เพื่อสร้างรายงาน)

การตรวจสอบเหล่านี้ต้องสแกนเนื้อหาที่เขียนจริง เรียกใช้สคริปต์ bash ที่เกี่ยวข้อง:

##### 2.5 การตรวจสอบความสอดคล้องของโลกทัศน์

รันคำสั่ง:
```bash
bash scripts/bash/check-world.sh --checklist
```

หากสคริปต์ไม่มี ให้แจ้งผู้ใช้ว่าฟังก์ชันนี้กำลังอยู่ในระหว่างการพัฒนา

##### 2.6 การตรวจสอบการจัดแนวเนื้อเรื่อง

รันคำสั่ง:
```bash
bash scripts/bash/check-plot.sh --checklist
```

##### 2.7 การตรวจสอบการซิงโครไนซ์ข้อมูล

รันคำสั่ง:
```bash
bash scripts/bash/check-consistency.sh --checklist
```

##### 2.8 การตรวจสอบเส้นเวลา

รันคำสั่ง:
```bash
bash scripts/bash/check-timeline.sh check --checklist
```

##### 2.9 การตรวจสอบสถานะการเขียน

รันคำสั่ง:
```bash
bash scripts/bash/check-writing-state.sh --checklist
```

### 3. สร้างผลลัพธ์ Checklist

**ตำแหน่งบันทึก**: `spec/checklists/`

**กฎการตั้งชื่อไฟล์**:
- ประเภทคุณภาพข้อกำหนด: `[ประเภท]-quality.md` (เช่น `outline-quality.md`)
- ประเภทการตรวจสอบเนื้อหา: `[ประเภท]-[วันที่].md` (เช่น `world-consistency-20251011.md`)

**รูปแบบผลลัพธ์**: ใช้ `templates/checklist-template.md` เป็นเทมเพลต

### 4. รายงานผลลัพธ์

แสดง:
- เส้นทางไฟล์ Checklist
- จำนวนรายการตรวจสอบทั้งหมด
- ประเภทและขอบเขตการตรวจสอบ
- คำอธิบายวิธีการใช้ Checklist

## ตัวอย่างการใช้งาน

```bash
# การตรวจสอบคุณภาพข้อกำหนด
/checklist คุณภาพโครงเรื่อง
/checklist การตั้งค่าตัวละคร
/checklist โลกทัศน์

# การตรวจสอบเนื้อหา
/checklist ความสอดคล้องของโลกทัศน์
/checklist การจัดแนวเนื้อเรื่อง
/checklist การซิงโครไนซ์ข้อมูล
```

## ข้อควรระวัง

1. **Checklist ประเภทคุณภาพข้อกำหนด**: ใช้สำหรับการตรวจสอบการวางแผนก่อนเขียน ค้นหาปัญหาคุณภาพของเอกสารเอง
2. **Checklist ประเภทการตรวจสอบเนื้อหา**: ใช้สำหรับการตรวจสอบเนื้อหาหลังเขียน ค้นหาปัญหาของผลลัพธ์จริง
3. Checklist ทั้งสองประเภทเสริมกัน แนะนำ: ใช้ประเภทแรกในช่วงวางแผน ใช้ประเภทที่สองในช่วงเขียน
4. Checklist ทั้งหมดบันทึกใน `spec/checklists/` เพื่อให้สามารถติดตามประวัติการตรวจสอบได้

## หมายเหตุความเข้ากันได้แบบย้อนหลัง

คำสั่งเก่า `/world-check` และ `/plot-check` ยังคงใช้งานได้ แต่แนะนำให้ใช้คำสั่ง `/checklist` แบบรวมแทน
