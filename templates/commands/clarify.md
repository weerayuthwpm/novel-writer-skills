## เป้าหมาย

ตรวจจับและลดความคลุมเครือหรือจุดตัดสินใจที่ขาดหายไปในโครงเรื่อง ผ่านการถาม-ตอบแบบโต้ตอบเพื่อรวบรวมข้อมูลให้กระจ่าง และบันทึกผลลัพธ์ไว้ในไฟล์เรื่อง

**หมายเหตุ**: กระบวนการให้ความกระจ่างนี้ควรดำเนินการและเสร็จสิ้นก่อน `/plan` หากผู้ใช้ระบุชัดเจนว่าข้ามการให้ความกระจ่าง (เช่น การเขียนเชิงสำรวจ) ก็สามารถดำเนินการต่อได้ แต่ต้องเตือนว่าความเสี่ยงในการทำงานซ้ำในขั้นตอนถัดไปจะเพิ่มขึ้น

## ขั้นตอนการดำเนินงาน

### 2. การสแกนความคลุมเครือแบบมีโครงสร้าง

ทำการสแกนโครงเรื่องอย่างครอบคลุม ประเมินความชัดเจนในแต่ละหมวดหมู่ (ชัดเจน/ชัดเจนบางส่วน/ขาดหาย):

**การวางตำแหน่งเชิงสร้างสรรค์**
- กลุ่มผู้อ่านเป้าหมาย (ช่วงอายุ แนวโน้มทางเพศ ระดับการอ่าน)
- การวางตำแหน่งผลงาน (นิยายบันเทิงเชิงพาณิชย์/วรรณกรรมจริงจัง/นวนิยายแนว)
- ขนาดที่คาดหวัง (สั้น 3-5 หมื่นคำ/กลาง 1-2 แสนคำ/ยาว 5 แสนคำขึ้นไป)

**การตั้งค่าโลก**
- ความแม่นยำของภูมิหลังยุคสมัย (ปี/ราชวงศ์/ระดับสมมติ)
- กฎของโลก (ระบบเวทมนตร์/ระดับเทคโนโลยี/ระบบสังคม)
- ขอบเขตทางภูมิศาสตร์ (เมืองเดียว/หลายประเทศ/ทวีป/อวกาศ)

**การออกแบบตัวละคร**
- เส้นโค้งการเติบโตของตัวละครหลัก (อ่อนแอแล้วเก่ง/อัจฉริยะ/พัฒนาอย่างมั่นคง)
- น้ำเสียงบุคลิกของตัวละครหลัก (เลือดร้อน/สงบ/เจ้าเล่ห์/บริสุทธิ์)
- บทบาทหน้าที่ของตัวละครรอง (ขับเคลื่อนเนื้อเรื่อง/สนับสนุนทางอารมณ์/เปรียบเทียบ/คอนทราสต์)
- ระดับสติปัญญาของตัวร้าย (โง่/สูสี/ล้มเหลวระดับสูง)

**กลยุทธ์การเล่าเรื่อง**
- การเลือกมุมมอง (บุรุษที่หนึ่ง/บุรุษที่สามจำกัด/มุมมองพระเจ้า)
- โครงสร้างเส้นเวลา (การเล่าแบบเส้นตรง/ย้อนอดีต/สลับเส้นหลายเส้น)
- จังหวะการเล่า (บันเทิงเร็ว/ลางสังหรณ์ช้า/ขึ้นลงเหมาะสม)

**แกนหลักของเนื้อเรื่อง**
- ประเภทความขัดแย้งหลัก (คน vs คน/คน vs ธรรมชาติ/คน vs สังคม/คน vs ตัวเอง)
- ความชัดเจนของเป้าหมายหลัก (แก้แค้น/เติบโต/ช่วยเหลือ/สำรวจ)
- แนวโน้มตอนจบ (สุขสมบูรณ์/โศกนาฏกรรม/เปิดจบ)

**น้ำเสียงและสไตล์**
- การเลือกสำนวน (คล่องแคล่ว/โบราณหรูหรา/ขบขัน/สมจริงเคร่งขรึม)
- การเน้นการพรรณนา (ฉากแอ็คชั่น/พรรณนาจิตวิทยา/บรรยากาศแวดล้อม/บทสนทนา)
- น้ำเสียงทางอารมณ์ (หลงใหล/หดหู่มืดมน/อบอุ่นฮีลใจ/สะเทือนใจ)

**ข้อจำกัดเชิงสร้างสรรค์**
- การจัดการเนื้อหาที่ละเอียดอ่อน (ระดับความรุนแรง/ขอบเขตอารมณ์)
- แนวทางค่านิยม (พลังบวก/สัจนิยม/วิพากษ์)
- แผนการอัปเดต (รายวัน/รายสัปดาห์/รายเดือน)

สำหรับแต่ละหมวดหมู่ที่ "ชัดเจนบางส่วน" หรือ "ขาดหาย" ให้สร้างคำถามที่อาจถามได้ เว้นแต่:
- การให้ความกระจ่างจะไม่ส่งผลต่อทิศทางการสร้างสรรค์อย่างมีนัยสำคัญ
- ข้อมูลเหมาะสมที่จะกำหนดในขั้นตอนการวางแผนบทมากกว่า

### 3. สร้างคิวคำถามลำดับความสำคัญ

สร้างคิวคำถามให้ความกระจ่างลำดับความสำคัญสูงสุดภายใน 5 ข้อ โดยใช้ข้อจำกัดดังนี้:
- ทั้งเซสชันมีจำนวนคำถามสูงสุด 5 ข้อ
- แต่ละคำถามต้องสามารถตอบได้ด้วยวิธีใดวิธีหนึ่งดังนี้:
  * แบบเลือกตอบ (2-5 ตัวเลือกที่ไม่ซ้ำกัน)
  * แบบตอบสั้น (จำกัด 5 คำ)
- ให้รวมเฉพาะคำถามที่มีผลกระทบอย่างมีนัยสำคัญต่อทิศทางการสร้างสรรค์
- รับประกันความสมดุลในการครอบคลุมหมวดหมู่ โดยให้ความสำคัญกับสาขาที่มีผลกระทบสูงก่อน
- หากมีหมวดหมู่ที่ต้องให้ความกระจ่างมากกว่า 5 ให้เลือกหมวดหมู่ที่มี (ผลกระทบ × ความไม่แน่นอน) สูงสุด 5 อันดับ

### 3.5 หลักการออกแบบคำถาม (ความเข้าใจแบบสนทนา)

แต่ละคำถามควรเป็น **เสมือนนักเขียนตัวจริงกำลังพูดคุยกัน** ไม่ใช่วิศวกรกรอกแบบฟอร์ม

**หลักการสำคัญ**:
- ✅ **บอกสิ่งที่สังเกตก่อน แล้วค่อยถาม**: ทำให้ผู้เขียนเข้าใจว่า "ทำไมถึงต้องถามเรื่องนี้"
- ✅ **ใช้ภาษาคน**: ภาษาของนักสร้างสรรค์ ไม่ใช่ศัพท์วิชาการ
- ✅ **ชี้ให้เห็นผลกระทบ**: ให้ผู้เขียนเข้าใจว่าการตัดสินใจนี้ส่งผลต่ออะไร
- ❌ **หลีกเลี่ยงการถามตรง ๆ**: อย่าเริ่มด้วย "กลุ่มผู้อ่านเป้าหมายของคุณคือ?"

**ตัวอย่างเปรียบเทียบ**:

❌ **การถามแบบวิศวกรรม** (ควรหลีกเลี่ยง):
```
คำถาม 1: กลุ่มผู้อ่านเป้าหมายของคุณคือช่วงอายุใด?
A. 18-25 ปี  B. 26-35 ปี  C. 36-45 ปี
```

✅ **การถามแบบสนทนา** (แนะนำ):
```
💬 ฉันสังเกตว่าเรื่องของคุณมีองค์ประกอบของโรงเรียน แต่ก็มีเนื้อหาเกี่ยวกับการทำงาน กลุ่มผู้อ่านของสองฉากนี้ต่างกันมาก——
ผู้อ่านวัยเรียนชอบการเติบโตแบบหลงใหล ส่วนผู้อ่านวัยทำงานให้ความสำคัญกับเกมการเมืองมากกว่า ซึ่งจะส่งผลโดยตรงต่อการออกแบบจังหวะและการแสดงค่านิยมของเรา

เลยอยากถามก่อนว่า: **คุณอยากเขียนให้ใครอ่านเป็นหลัก?**

| ตัวเลือก | คำอธิบาย |
|------|------|
| A | กลุ่มนักเรียน (18-25 ปี) - เน้นการเติบโตและอุดมคติ |
| B | กลุ่มวัยทำงาน (26-35 ปี) - เน้นความจริงและกลยุทธ์ |
| C | ครอบคลุมทุกกลุ่ม (ปรับเป็นการเล่าแบบสองเส้น พิจารณาทั้งสอง)|
| D | กำหนดเอง (กรุณาใส่ความคิดของคุณ)|
```

**โครงสร้างเทมเพลตการถาม**:
```markdown
💬 [สิ่งที่สังเกตได้/ประเด็นที่ขัดแย้ง] [สิ่งนี้จะส่งผลต่อการตัดสินใจเชิงสร้างสรรค์อะไร]

เลยอยากถามว่า: **[คำถามหลัก]**

[ตารางตัวเลือก หรือข้อความให้ตอบสั้น]
```

### 4. วงจรการถาม-ตอบตามลำดับ

**แสดงทีละหนึ่งคำถาม** ตามรูปแบบการถามแบบสนทนา

รูปแบบคำถามแบบเลือกตอบ (ต้องมีบริบทของคำถาม):
```markdown
💬 [บริบทคำถาม: คุณสังเกตเห็นอะไร? สิ่งนี้ส่งผลต่ออะไร?]

เลยอยากถามว่า: **[คำถามหลัก]**

| ตัวเลือก | คำอธิบาย |
|------|------|
| A | รายละเอียดตัวเลือก A |
| B | รายละเอียดตัวเลือก B |
| C | รายละเอียดตัวเลือก C |
| D | รายละเอียดตัวเลือก D |
| E | รายละเอียดตัวเลือก E (ไม่บังคับ) |
| F | กำหนดเอง (กรุณาใส่ความคิดของคุณ)|
```

รูปแบบคำถามแบบตอบสั้น (ต้องมีบริบทของคำถาม):
```markdown
💬 [บริบทคำถาม: คุณสังเกตเห็นอะไร? สิ่งนี้ส่งผลต่ออะไร?]

เลยอยากถามว่า: **[คำถามหลัก]**

กรุณาตอบสั้น ๆ (ไม่เกิน 5 คำ): _______
```

**การจัดการคำตอบของผู้ใช้**
- ตรวจสอบความถูกต้องของคำตอบ
- หากเลือก F (กำหนดเอง) ให้รับและบันทึกเนื้อหาที่ผู้ใช้ป้อนเอง
- หากมีความคลุมเครือ ให้ขอให้ผู้ใช้ช่วยชี้แจงอย่างรวดเร็ว
- บันทึกคำตอบและไปยังคำถามถัดไป

**เงื่อนไขการหยุด**
- ความคลุมเครือที่สำคัญทั้งหมดได้รับการแก้ไขแล้ว
- ผู้ใช้ส่งสัญญาณเสร็จ ("พอแล้ว" "เพียงพอ" "ไม่ต้องแล้ว")
- ถึงขีดจำกัดสูงสุด 5 คำถามแล้ว

### 5. รวบรวมผลลัพธ์การให้ความกระจ่าง

หลังแต่ละคำตอบที่รับได้ ให้ดำเนินการทันที:

**เมื่อทำการรวบรวมครั้งแรก**
- สร้างหัวข้อ `## บันทึกการให้ความกระจ่าง` ในโครงเรื่อง (ถ้ายังไม่มี)
- เพิ่มหัวข้อย่อย `### เซสชันการให้ความกระจ่าง [วันที่]`

**รูปแบบการบันทึก**
```markdown
- ถาม: [เนื้อหาคำถาม] → ตอบ: [คำตอบผู้ใช้]
```

**อัปเดตหัวข้อที่เกี่ยวข้อง**
ตามเนื้อหาที่ให้ความกระจ่าง ให้ปรับปรุงส่วนที่เกี่ยวข้องของโครงเรื่อง:
- การวางตำแหน่งเชิงสร้างสรรค์ → อัปเดตภาพรวมเรื่อง
- โลกทัศน์ → อัปเดตการตั้งค่าโลก
- ตัวละคร → อัปเดตการตั้งค่าตัวละคร
- กลยุทธ์การเล่าเรื่อง → เพิ่มในคำอธิบายการสร้างสรรค์
- น้ำเสียงและสไตล์ → เพิ่มในแนวทางสไตล์

### 6. ตรวจสอบและบันทึก

หลังการอัปเดตแต่ละครั้งให้ตรวจสอบ:
- ความสมบูรณ์ของบันทึกการให้ความกระจ่าง
- ไม่มีเครื่องหมายคลุมเครือที่ค้างอยู่ซึ่งได้รับการแก้ไขแล้ว
- ไม่มีข้อความที่ขัดแย้งกัน
- รูปแบบ Markdown ถูกต้อง

เขียนเนื้อหาที่อัปเดตกลับไปยังไฟล์เรื่อง

### 7. รายงานสรุป

รายงานประกอบด้วย:
- จำนวนคำถามที่ถามและตอบ
- เส้นทางไฟล์เรื่องที่อัปเดต
- รายการหัวข้อที่แตะต้อง
- ตารางสรุปความครอบคลุม:

| หมวดหมู่ | สถานะ |
|------|------|
| การวางตำแหน่งเชิงสร้างสรรค์ | ✅ ให้ความกระจ่างแล้ว |
| การตั้งค่าโลก | ✅ ให้ความกระจ่างแล้ว |
| การออกแบบตัวละคร | ⏸ เลื่อนไปที่การวางแผน |
| ... | ... |

- คำสั่งถัดไปที่แนะนำ (โดยปกติคือ `/plan`)

## กฎการปฏิบัติ

- หากไม่พบความคลุมเครือที่มีนัยสำคัญ ให้ตอบ: "ไม่พบความคลุมเครือสำคัญที่ต้องให้ความกระจ่างทันที"
- หากไฟล์เรื่องหายไป ให้แนะนำให้ผู้ใช้รัน `/story` ก่อน
- ไม่เกินขีดจำกัด 5 คำถาม
- หลีกเลี่ยงการถามรายละเอียดทางเทคนิคการเขียนล้วน ๆ
- เคารพสัญญาณหยุดก่อนกำหนดของผู้ใช้
- หากบรรลุความครอบคลุมสมบูรณ์โดยไม่ต้องถาม ให้แสดงสรุปความครอบคลุมแบบกระชับ

## ข้อพิจารณาเฉพาะสำหรับการเขียนนิยาย

- **การปรับตามประเภท**: ตามประเภทของเรื่อง (บันเทิง/ระทึกขวัญ/รัก/วรรณกรรมจริงจัง ฯลฯ) โหลดฐานความรู้ที่เกี่ยวข้องเพื่อให้คำถามที่ตรงจุด
- **การเน้นผู้อ่าน**: มาตรฐานและจุดเน้นที่ต่างกันระหว่างผลงานเชิงพาณิชย์และผลงานวรรณกรรม
- **ความอ่อนไหวทางวัฒนธรรม**: บางเนื้อหาต้องได้รับการจัดการอย่างระมัดระวังเป็นพิเศษ
- **การวางแผนซีรีส์**: การเป็นงานซีรีส์หรือไม่ส่งผลต่อสถาปัตยกรรมโดยรวมและการตัดสินใจ
