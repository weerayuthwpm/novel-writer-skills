# การติดตามความคืบหน้าแบบครบวงจร

แสดงสถานะและความคืบหน้าในด้านต่างๆ ของการเขียนนิยายอย่างครอบคลุม

## มิติในการติดตาม

1. **ความคืบหน้าการเขียน** - จำนวนคำ, บท, อัตราความสำเร็จ
2. **พัฒนาการของเนื้อเรื่อง** - ความคืบหน้าของเส้นเรื่องหลัก, สถานะของเส้นเรื่องรอง
3. **เส้นเวลา** - การดำเนินไปของเวลาในเรื่อง
4. **สถานะตัวละคร** - พัฒนาการและตำแหน่งของตัวละคร
5. **การจัดการปม** - สถานะการวางปมและการเก็บปม

## วิธีการใช้งาน

รันสคริปต์ {SCRIPT} [ตัวเลือก]:
- ไม่มีพารามิเตอร์ - แสดงรายงานการติดตามฉบับสมบูรณ์
- `--brief` - แสดงข้อมูลโดยสังเขป
- `--plot` - แสดงเฉพาะการติดตามเนื้อเรื่อง
- `--stats` - แสดงเฉพาะข้อมูลสถิติ
- `--check` - **[เพิ่มประสิทธิภาพ]** ดำเนินการตรวจสอบความสอดคล้องเชิงลึก (รวมถึงการตรวจสอบตัวละคร)
- `--fix` - **[ฟีเจอร์ใหม่]** ซ่อมแซมปัญหาง่าย ๆ ที่พบโดยอัตโนมัติ

## แหล่งข้อมูล

รวบรวมข้อมูลจากไฟล์ติดตามหลายแหล่ง:
- `progress.json` - ความคืบหน้าการเขียน
- `spec/tracking/plot-tracker.json` - การติดตามเนื้อเรื่อง
- `spec/tracking/timeline.json` - เส้นเวลา
- `spec/tracking/relationships.json` - เครือข่ายความสัมพันธ์
- `spec/tracking/character-state.json` - สถานะตัวละคร
- `spec/tracking/validation-rules.json` - **[ฟีเจอร์ใหม่]** กฎการตรวจสอบ (ใช้สำหรับ --check และ --fix)

## ฟังก์ชันเพิ่มประสิทธิภาพ

### การตรวจสอบความสอดคล้องของข้อมูล

การตรวจสอบพื้นฐาน (ดำเนินการโดยค่าเริ่มต้น):
- ความสอดคล้องระหว่าง plot-tracker.json กับ outline.md
- ตรรกะเวลาของ timeline.json
- ความขัดแย้งของความสัมพันธ์ใน relationships.json
- ความสมเหตุสมผลของตำแหน่งใน character-state.json

### โหมดการตรวจสอบเชิงลึก (--check)

เมื่อใช้พารามิเตอร์ `--check` จะดำเนินการตรวจสอบเชิงลึกตามโปรแกรม:

#### ขั้นตอนการทำงานภายใน (ดำเนินการอัตโนมัติ)
```markdown

- [x] T001 [P] ตรวจสอบความสอดคล้องของเนื้อเรื่อง (ตรรกะ plot-check)
- [x] T002 [P] ตรวจสอบเส้นเวลา (ตรรกะ timeline)
- [x] T003 [P] ตรวจสอบความสัมพันธ์ (ตรรกะ relations)
- [x] T004 [P] ตรวจสอบโลกทัศน์ (ตรรกะ world-check)

# ระยะที่ 2: การตรวจสอบตัวละครเชิงลึก
- [x] T005 โหลดกฎการตรวจสอบจาก validation-rules.json
- [x] T006 สแกนชื่อตัวละครในทุกบท
- [x] T007 เปรียบเทียบกับ character-state.json เพื่อตรวจสอบความสอดคล้องของชื่อ
- [x] T008 ตรวจสอบการเรียกชื่อตาม relationships.json
- [x] T009 ตรวจสอบพฤติกรรมตัวละครว่าสอดคล้องกับบุคลิกหรือไม่

# ระยะที่ 3: สร้างรายงานฉบับสมบูรณ์
- [x] T010 รวบรวมผลการตรวจสอบทั้งหมด
- [x] T011 ทำเครื่องหมายระดับความรุนแรงของปัญหา
- [x] T012 สร้างคำแนะนำในการซ่อมแซม
```

### โหมดการซ่อมแซมอัตโนมัติ (--fix)

เมื่อใช้พารามิเตอร์ `--fix` จะดำเนินการซ่อมแซมตามรายงานการตรวจสอบโดยอัตโนมัติ:

#### ขอบเขตการซ่อมแซมอัตโนมัติ
1. **ชื่อตัวละครผิดพลาด** - แทนที่โดยอัตโนมัติตาม validation-rules.json
2. **การเรียกชื่อคงที่ผิดพลาด** - แก้ไขเป็นคำเรียกที่ถูกต้องโดยอัตโนมัติ
3. **การสะกดผิดเล็กน้อย** - แก้ไขข้อผิดพลาดที่เห็นได้ชัด

#### ขั้นตอนการซ่อมแซม
```markdown
# งานซ่อมแซมภายใน (ดำเนินการอัตโนมัติ)
- [x] F001 อ่านรายการปัญหาจากรายงานการตรวจสอบ
- [x] F002 [P] ซ่อมแซมชื่อตัวละครในบทที่ 3
- [x] F003 [P] ซ่อมแซมการเรียกชื่อผิดในบทที่ 7
- [x] F004 สร้างรายงานการซ่อมแซม
- [x] F005 อัปเดตไฟล์ติดตาม
```

#### ตัวอย่างรายงานการซ่อมแซม
```
🔧 รายงานการซ่อมแซมอัตโนมัติ
━━━━━━━━━━━━━━━━━━━━━
✅ ซ่อมแซมแล้ว: 2 รายการ
- บทที่ 3: "หลี่หมิง" → "หลี่จงยง"
- บทที่ 7: "พี่ชาย" → "ท่านชาย"

⚠️ ต้องดำเนินการด้วยมนุษย์: 1 รายการ
- บทที่ 12: การกระโดดของเส้นเวลาต้องเพิ่มคำอธิบาย

ซ่อมแซมเสร็จสมบูรณ์! แนะนำให้รัน /track --check อีกครั้งเพื่อยืนยัน
```

### การวิเคราะห์อัจฉริยะและคำแนะนำ

1. **การวิเคราะห์ความคืบหน้า**
   - เปรียบเทียบแผนกับความคืบหน้าจริง
   - คาดการณ์เวลาที่จะเสร็จ
   - ระบุจุดติดขัดในการเขียน

2. **การวิเคราะห์เนื้อหา**
   - อัตราความครอบคลุมของปม (วางไปแล้ว / เก็บแล้ว)
   - ความถี่ในการปรากฏตัวของตัวละคร
   - เส้นโค้งความรุนแรงของความขัดแย้ง

3. **คำแนะนำในการดำเนินการ**
   จากผลการวิเคราะห์ให้:
   - จุดเน้นในการเขียนขั้นต่อไป
   - ปมที่ต้องจัดการ
   - สายความสัมพันธ์ที่ควรเสริม
   - คำแนะนำในการปรับเส้นเวลา

```
1. เก็บปม "กระจกสำริดโบราณ" ก่อนบทที่ 65
2. เสริมความขัดแย้งโดยตรงระหว่างตัวเอกกับผู้ร้าย
3. เพิ่มรายละเอียดเส้นเวลาในเล่มที่ 2

- ตัวละคร B ไม่ปรากฏตัวมา 5 บทแล้ว

- การกระโดดเวลาที่บทที่ 45 ต้องมีคำอธิบาย
```
