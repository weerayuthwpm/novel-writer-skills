# ระบบฐานความรู้การเขียน

## ภาพรวม

นี่คือขีดความสามารถหลักของ Novel Writer Skills — ระบบฐานความรู้การเขียนแบบทั่วไปที่สามารถขยายได้

**ความแตกต่างจากความรู้เฉพาะโครงการ**：
- `spec/knowledge/` - เก็บความรู้เฉพาะโครงการ (แฟ้มตัวละคร, สถานที่, โลกทัศน์ ฯลฯ)
- `templates/knowledge-base/` (ไดเรกทอรีนี้) - เก็บความรู้การเขียนแบบทั่วไป (แนวปฏิบัติ, เทคนิค, เอกสารอ้างอิง)

**หลักการทำงาน**：
1. **เปิดใช้งานอัตโนมัติ**：setting-detector Skill จะตรวจจับการตั้งค่าเรื่องราวโดยอัตโนมัติตามตารางการจับคู่คำสำคัญ
2. **โหลดตามความจำเป็น**：โหลดเฉพาะฐานความรู้ที่เกี่ยวข้อง ประหยัดโทเค็น (ฐานความรู้แต่ละแห่ง ~500 โทเค็น)
3. **ประยุกต์ใช้ต่อเนื่อง**：ฐานความรู้จะยังคงทำงานตลอดกระบวนการสร้างสรรค์

**ประสิทธิภาพของโทเค็น**：
- แผนเดิม (50 Skill)：~2000 โทเค็น
- แผนปัจจุบัน (detector 1 ตัว + ฐานความรู้ตามความจำเป็น)：~600 โทเค็น
- **ประหยัดได้ 75%**

## 📚 ดัชนีฐานความรู้

### 1. ฐานความรู้ประเภทนิยาย (Genres)

แนวปฏิบัติและแนวทางปฏิบัติที่ดีที่สุดสำหรับนวนิยายแนวอาชีพ

| ไฟล์ | ประเภท | แนวปฏิบัติสำคัญ |
|------|------|---------|
| `genres/romance.md` | นิยายรัก | ตอนจบแบบแฮปปี้ ฮีลลิ่ง, จังหวะอารมณ์, ส่วนโค้งความสัมพันธ์ |
| `genres/mystery.md` | สืบสวนสอบสวน | การเล่นที่ยุติธรรม, การจัดการเบาะแส, การออกแบบผู้ต้องสงสัย |
| `genres/historical.md` | นิยายอิงประวัติศาสตร์ | ความสมดุลของการค้นคว้า, บรรยากาศยุคสมัย, การเล่าแบบปล่อยภาพ |
| `genres/revenge.md` | แนวแก้แค้นสนุก | การควบคุมจังหวะ, การออกแบบการตบหน้า, การจัดการอารมณ์ |
| `genres/wuxia.md` | นิยายกำลังภายใน | กฎแห่งยุทธภพ, ระบบวิชายุทธ, จิตวิญญาณแห่งความกล้าหาญ |

### 2. ฐานความรู้เทคนิคการเขียน (Craft)

เทคนิคการเขียนมืออาชีพที่ใช้ได้ข้ามประเภท

| ไฟล์ | เทคนิค | หลักการสำคัญ |
|------|------|---------|
| `craft/dialogue.md` | เทคนิคบทสนทนา | ความหมายใต้คำพูด, เสียงของตัวละคร, ความเป็นธรรมชาติ |
| `craft/scene-structure.md` | โครงสร้างฉาก | เป้าหมาย-ความขัดแย้ง-หายนะ, แบบจำลองการต่อเนื่อง |
| `craft/character-arc.md` | ส่วนโค้งตัวละคร | กลไกการเปลี่ยนแปลง, ความขัดแย้งภายใน-ภายนอก, ตรรกะการเติบโต |
| `craft/pacing.md` | การควบคุมจังหวะ | การผ่อนคลายพอดี, การออกแบบไคลแม็กซ์, การจัดการช่วงเปลี่ยนผ่าน |
| `craft/show-not-tell.md` | เทคนิคการแสดง | รายละเอียดทางประสาทสัมผัส, การบอกใบ้ผ่านพฤติกรรม, หลีกเลี่ยงการสั่งสอน |

### 3. คลังเอกสารอ้างอิง (References)

ข้อมูลจริงเกี่ยวกับยุคสมัย วัฒนธรรม และภูมิหลังเฉพาะ

| ไดเรกทอรี | เนื้อหา | วัตถุประสงค์ |
|------|------|------|
| `references/china-1920s/` | จีนยุค 1920s | การต่อสู้ของขุนศึก, สภาพสังคม, ชีวิตประจำวัน |
| └─ `warlords.md` | ระบบขุนศึก | ความสัมพันธ์ระหว่างกลุ่ม, การจัดกำลังทหาร, โครงสร้างอำนาจ |
| └─ `society.md` | โครงสร้างสังคม | การแบ่งชนชั้น, ความแตกต่างเมือง-ชนบท, สภาพการศึกษา |
| └─ `daily-life.md` | ชีวิตประจำวัน | อาหาร เครื่องนุ่งห่ม ที่อยู่อาศัย การเดินทาง, เงินตรา ราคาสินค้า, วัฒนธรรมบันเทิง |

**การขยายในอนาคต**：
- `references/ancient-china/` - จีนโบราณแต่ละราชวงศ์
- `references/modern-workplace/` - สถานที่ทำงานสมัยใหม่
- `references/mythology/` - ระบบเทพปกรณัม
- ... (สามารถขยายได้ไม่จำกัด)

## 🔍 ตารางการจับคู่คำสำคัญ

**วัตถุประสงค์**：ใช้โดย Skill `setting-detector` เพื่อเปิดใช้งานฐานความรู้ที่เกี่ยวข้องโดยอัตโนมัติตามคำสำคัญที่ผู้ใช้ป้อน

### คำสำคัญฐานความรู้ประเภทนิยาย

```yaml
romance:
  keywords: [言情, 爱情, 恋爱, 浪漫, 感情线, 关系弧, CP, 甜文, 虐文]
  auto_load: genres/romance.md

mystery:
  keywords: [悬疑, 推理, 侦探, 破案, 谜团, 线索, 真相, 凶手, 犯罪]
  auto_load: genres/mystery.md

historical:
  keywords: [历史, 古代, 朝代, 考据, 时代背景, 历史小说]
  auto_load: genres/historical.md

revenge:
  keywords: [复仇, 报仇, 打脸, 爽文, 逆袭, 反击]
  auto_load: genres/revenge.md

wuxia:
  keywords: [武侠, 江湖, 武功, 侠客, 门派, 武学, 剑客]
  auto_load: genres/wuxia.md
```

### คำสำคัญเทคนิคการเขียน

```yaml
dialogue:
  triggers: [对话, 角色说话, 台词, 交流场景, 谈话]
  auto_load: craft/dialogue.md

scene-structure:
  triggers: [场景, 章节结构, 情节推进, 场景设计]
  auto_load: craft/scene-structure.md

character-arc:
  triggers: [角色成长, 人物弧线, 角色改变, 性格转变]
  auto_load: craft/character-arc.md

pacing:
  triggers: [节奏, 快慢, 拖沓, 高潮, 张力]
  auto_load: craft/pacing.md

show-not-tell:
  triggers: [展示, 具象化, 说教, 感官细节, 描写]
  auto_load: craft/show-not-tell.md
```

### คำสำคัญเอกสารอ้างอิง

```yaml
china-1920s:
  keywords: [1920, 民国, 军阀, 北洋, 穿越民国, 二十年代]
  auto_load:
    - references/china-1920s/warlords.md
    - references/china-1920s/society.md
    - references/china-1920s/daily-life.md
```

## 🛠️ คู่มือการใช้งาน

### ขั้นตอนการเปิดใช้งานอัตโนมัติ

1. **ผู้ใช้กล่าวถึงคำสำคัญ**：
   ```
   ผู้ใช้："ฉันจะเขียนนิยายรักแนวแก้แค้นในยุค 1920s"
   ```

2. **setting-detector ตรวจจับ**：
   - ตรวจจับ "1920" → เปิดใช้งาน `references/china-1920s/`
   - ตรวจจับ "言情" (รัก) → เปิดใช้งาน `genres/romance.md`
   - ตรวจจับ "复仇" (แก้แค้น) → เปิดใช้งาน `genres/revenge.md`

3. **โหลดตามความจำเป็น**：
   ```
   ✓ โหลดแล้ว genres/romance.md (520 โทเค็น)
   ✓ โหลดแล้ว genres/revenge.md (480 โทเค็น)
   ✓ โหลดแล้ว references/china-1920s/*.md (650 โทเค็น)

   รวม：~1650 โทเค็น (โหลดเฉพาะความรู้ที่เกี่ยวข้อง)
   ```

4. **ประยุกต์ใช้ต่อเนื่อง**：
   - ใน `/specify`：แนะนำองค์ประกอบสำคัญ
   - ใน `/plan`：เสนอกรอบโครงสร้าง
   - ใน `/write`：คำแนะนำการเขียนแบบเรียลไทม์
   - ใน `/analyze`：ตรวจสอบแนวปฏิบัติประเภท

### การระบุโหลดด้วยตนเอง

ถ้าการตรวจจับอัตโนมัติล้มเหลว ผู้ใช้สามารถระบุได้อย่างชัดเจน：

```
"กรุณาโหลดฐานความรู้ romance และ mystery"
"เรื่องนี้ต้องการเอกสารอ้างอิงจีน 1920s"
```

### ดูฐานความรู้ที่เปิดใช้งานอยู่ในปัจจุบัน

ผู้ใช้สามารถถามได้ทุกเมื่อ：
```
"ฐานความรู้ใดที่เปิดใช้งานอยู่ในปัจจุบัน?"
```

AI จะตอบ：
```
📚 ฐานความรู้ที่เปิดใช้งานอยู่ในปัจจุบัน：
✓ genres/romance.md - แนวปฏิบัตินิยายรัก
✓ genres/revenge.md - เทคนิคนิยายแก้แค้นสนุก
✓ references/china-1920s/ - ข้อมูลจีนยุค 1920s
```

## 📦 การขยายฐานความรู้

### การเพิ่มฐานความรู้ใหม่

1. **กำหนดหมวดหมู่**：genres / craft / references

2. **สร้างไฟล์**：
   ```bash
   # ความรู้ประเภท
   touch templates/knowledge-base/genres/sci-fi.md

   # เทคนิคการเขียน
   touch templates/knowledge-base/craft/worldbuilding.md

   # เอกสารอ้างอิง
   mkdir -p templates/knowledge-base/references/ancient-rome
   touch templates/knowledge-base/references/ancient-rome/military.md
   ```

3. **อัปเดตตารางการจับคู่**：เพิ่มรายการที่เกี่ยวข้องในตารางการจับคู่คำสำคัญของ README นี้

4. **เขียนเนื้อหา**：อ้างอิงรูปแบบของฐานความรู้ที่มีอยู่ (500-800 บรรทัด)

### รูปแบบข้อกำหนดของฐานความรู้

ไฟล์ฐานความรู้แต่ละไฟล์ควรประกอบด้วย：

```markdown
# [ชื่อฐานความรู้]

## ข้อมูลอ้างอิงด่วน (Quick Reference)
[บทสรุป 1-3 ย่อหน้า อธิบายหลักการสำคัญ]

## หลักการสำคัญ (Core Principles)
[กฎสำคัญ 3-5 ข้อ พร้อมคำอธิบายโดยละเอียด]

## การประยุกต์ใช้ในทางปฏิบัติ (Practical Application)
[วิธีการประยุกต์ใช้ความรู้นี้ในแต่ละขั้นตอนการสร้างสรรค์]

## ข้อผิดพลาดที่พบบ่อย (Common Pitfalls)
[ข้อผิดพลาดที่มือใหม่ในประเภทนี้มักทำและวิธีหลีกเลี่ยง]

## การวิเคราะห์ตัวอย่าง (Examples)
[การวิเคราะห์กรณีศึกษาจากผลงานคลาสสิก]

## การบูรณาการกับคำสั่ง (Integration with Commands)
[วิธีการประยุกต์ใช้ใน /specify, /plan, /write, /analyze]
```

### การแยกฐานความรู้จากนวนิยายคลาสสิก

ในอนาคตสามารถแยกวิเคราะห์นวนิยายคลาสสิกเพื่อสร้างเป็นฐานความรู้ได้：

```bash
# ตัวอย่าง：แยกความรู้แนววิทยาศาสตร์แข็งจาก《สามตัว》
templates/knowledge-base/genres/hard-sci-fi.md

# แยกความรู้การเล่าเรื่องครอบครัวจาก《ความฝันในหอแดง》
templates/knowledge-base/craft/family-saga.md
```

## 📊 สถานะฐานความรู้

**สถานะปัจจุบัน**：เวอร์ชันเริ่มต้น

| หมวดหมู่ | เสร็จแล้ว | อยู่ในแผน | รวม |
|------|--------|--------|------|
| ความรู้ประเภท | 5 | 10+ | 15+ |
| เทคนิคการเขียน | 5 | 8+ | 13+ |
| เอกสารอ้างอิง | 1 | 20+ | 21+ |

**แผนการขยายขั้นต่อไป**：
- [ ] นิยายแฟนตาซี (fantasy)
- [ ] นิยายวิทยาศาสตร์ (sci-fi)
- [ ] นิยายสยองขวัญ (horror)
- [ ] เทคนิคการสร้างโลก (worldbuilding)
- [ ] เอกสารอ้างอิงจีนโบราณแต่ละราชวงศ์
- [ ] เอกสารอ้างอิงเมืองสมัยใหม่
