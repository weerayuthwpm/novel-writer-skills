# ระบบคลังความรู้สำหรับการเขียน (Writing Knowledge Base System)

## ภาพรวม

นี่คือความสามารถหลักที่โดดเด่นของ Novel Writer Skills — ระบบคลังความรู้ทั่วไปสำหรับการเขียนที่สามารถขยายขีดความสามารถได้[cite: 2]

**ความแตกต่างระหว่างความรู้เฉพาะเจาะจงของโครงการ (Project-Specific Knowledge)**:
- `spec/knowledge/` — ใช้จัดเก็บความรู้เฉพาะเจาะจงของโครงการ (เช่น ข้อมูลตัวละคร, สถานที่, โลกทัศน์/เซตติง เป็นต้น)[cite: 2]
- `templates/knowledge-base/` (ไดเรกทอรีนี้) — ใช้จัดเก็บความรู้ทั่วไปสำหรับการเขียน (เช่น ขนบนิยายแต่ละประเภท, เทคนิคการเขียน, ข้อมูลอ้างอิง)[cite: 2]

**หลักการทำงาน**:
1. **เปิดใช้งานอัตโนมัติ**: สกิล `setting-detector` จะตรวจจับเซตติงของเรื่องราวโดยอัตโนมัติตามตารางจับคู่คำสำคัญ (Keyword Mapping)[cite: 2]
2. **โหลดข้อมูลตามความต้องการ (On-Demand)**: โหลดเฉพาะคลังความรู้ที่เกี่ยวข้องเพื่อประหยัด Token (คลังความรู้แต่ละแห่งใช้ประมาณ ~500 tokens)[cite: 2]
3. **ประยุกต์ใช้อย่างต่อเนื่อง**: คลังความรู้จะยังคงเปิดใช้งานอยู่ตลอดกระบวนการสร้างสรรค์ผลงาน[cite: 2]

**ประสิทธิภาพของ Token**:
- รูปแบบเดิม (ใช้ 50 สกิล): ~2,000 tokens[cite: 2]
- รูปแบบปัจจุบัน (ใช้ 1 detector + คลังความรู้ตามความต้องการ): ~600 tokens[cite: 2]
- **ประหยัดได้ถึง 75%**[cite: 2]

## 📚 ดัชนีคลังความรู้

### 1. คลังความรู้ประเภทนิยาย (Genres)

ขนบการเขียนและแนวทางปฏิบัติที่ดีที่สุดสำหรับนิยายเฉพาะทางแต่ละประเภท[cite: 2]

| ไฟล์ | ประเภทนิยาย | ขนบและข้อตกลงสำคัญ |
|------|------|---------|
| `genres/romance.md` | นิยายรัก/โรแมนติก | ตอนจบแบบ HEA (Happy Ever After), จังหวะทางอารมณ์, พัฒนาการความสัมพันธ์ (Relationship Arc)[cite: 2] |
| `genres/mystery.md` | นิยายสืบสวนสอบสวน | การดำเนินเนื้อเรื่องแบบยุติธรรม (Fair Play), การจัดการเบาะแส, การออกแบบผู้ต้องสงสัย[cite: 2] |
| `genres/historical.md` | นิยายอิงประวัติศาสตร์ | สมดุลของการตรวจสอบข้อมูล, บรรยากาศยุคสมัย, การพรรณนาเชิงประวัติศาสตร์[cite: 2] |
| `genres/revenge.md` | นิยายแนวแก้แค้น/สะใจ | การควบคุมจังหวะเรื่อง, การออกแบบฉากตบหน้า/โชว์เหนือ, การจัดการอารมณ์[cite: 2] |
| `genres/wuxia.md` | นิยายกำลังภายใน | กฎเกณฑ์ชาวยุทธ, ระบบวรยุทธ, จิตวิญญาณแห่งความจอมยุทธ์[cite: 2] |

### 2. คลังความรู้เทคนิคการเขียน (Craft)

เทคนิคการเขียนระดับมืออาชีพที่สามารถประยุกต์ใช้ได้ข้ามประเภทนิยาย[cite: 2]

| ไฟล์ | เทคนิค | หลักการสำคัญ |
|------|------|---------|
| `craft/dialogue.md` | เทคนิคการเขียนบทสนทนา | นัยแฝง (Subtext), เสียงของตัวละคร (Voice), ความเป็นธรรมชาติ[cite: 2] |
| `craft/scene-structure.md` | โครงสร้างฉาก | เป้าหมาย-ความขัดแย้ง-ความหายนะ (Goal-Conflict-Disaster), โมเดลฉากต่อเนื่อง (Sequel Model)[cite: 2] |
| `craft/character-arc.md` | พัฒนาการตัวละคร | กลไกความเปลี่ยนแปลง, ความขัดแย้งภายในและภายนอก, ตรรกะการเติบโต[cite: 2] |
| `craft/pacing.md` | การควบคุมจังหวะเรื่อง | ความยืดหยุ่นผ่อนหนักผ่อนเบา, การออกแบบจุดไคลแมกซ์, การจัดการช่วงเปลี่ยนผ่าน[cite: 2] |
| `craft/show-not-tell.md` | เทคนิคการแสดงให้เห็น (Show, Don't Tell) | รายละเอียดทางประสาทสัมผัส, การบอกใบ้ผ่านพฤติกรรม, หลีกเลี่ยงการสั่งสอน/บรรยายตรงๆ[cite: 2] |

### 3. คลังข้อมูลอ้างอิง (References)

ข้อมูลจริงตามยุคสมัย วัฒนธรรม และภูมิหลังเฉพาะ[cite: 2]

| ไดเรกทอรี | เนื้อหา | ประโยชน์การใช้งาน |
|------|------|------|
| `references/china-1920s/` | ประเทศจีนในทศวรรษ 1920 | สงครามขุนศึก, สภาพสังคม, ชีวิตประจำวัน[cite: 2] |
| └─ `warlords.md` | ระบบขุนศึก | ความสัมพันธ์ระหว่างฝ่าย, การจัดโครงสร้างกองทัพ, โครงสร้างอำนาจ[cite: 2] |
| └─ `society.md` | โครงสร้างสังคม | การแบ่งแยกชนชั้น, ความแตกต่างระหว่างเมืองและชนบท, สภาพการศึกษา[cite: 2] |
| └─ `daily-life.md` | ชีวิตประจำวัน | เครื่องนุ่งห่ม อาหาร ที่อยู่อาศัย การเดินทาง, ค่าเงินและราคาสินค้า, วัฒนธรรมความบันเทิง[cite: 2] |

**การขยายขีดความสามารถในอนาคต**:
- `references/ancient-china/` — ประเทศจีนโบราณยุคราชวงศ์ต่างๆ[cite: 2]
- `references/modern-workplace/` — แวดวงการทำงานในยุคปัจจุบัน[cite: 2]
- `references/mythology/` — ระบบตำนานและเทพปกรณัม[cite: 2]
- ... (สามารถขยายขีดความสามารถได้ไม่จำกัด)[cite: 2]

## 🛠️ คู่มือการใช้งาน

### ขั้นตอนการเปิดใช้งานอัตโนมัติ

1. **เมื่อผู้ใช้ระบุคำสำคัญ**:



```
   ผู้ใช้: "ฉันต้องการเขียนนิยายรักแนวแก้แค้นย้อนยุคในประเทศจีนช่วงทศวรรษ 1920"

```

2. **การตรวจจับของ setting-detector**:


* ตรวจพบ "1920" → เปิดใช้งาน `references/china-1920s/`

* ตรวจพบ "นิยายรัก" → เปิดใช้งาน `genres/romance.md`

* ตรวจพบ "แก้แค้น" → เปิดใช้งาน `genres/revenge.md`



3. **โหลดข้อมูลตามความต้องการ**:



```
   ✓ โหลดเรียบร้อย genres/romance.md (520 tokens)
   ✓ โหลดเรียบร้อย genres/revenge.md (480 tokens)
   ✓ โหลดเรียบร้อย references/china-1920s/*.md (650 tokens)

   รวมทั้งสิ้น: ~1650 tokens (โหลดเฉพาะความรู้ที่เกี่ยวข้องเท่านั้น)

```

4. **การประยุกต์ใช้อย่างต่อเนื่อง**:


* เมื่อใช้คำสั่ง `/specify`: แนะนำองค์ประกอบสำคัญ


* เมื่อใช้คำสั่ง `/plan`: มอบโครงสร้างและกรอบการดำเนินเรื่อง


* เมื่อใช้คำสั่ง `/write`: ให้คำแนะนำการเขียนแบบเรียลไทม์


* เมื่อใช้คำสั่ง `/analyze`: ตรวจสอบขนบประเภทนิยาย

```

"เรื่องนี้จำเป็นต้องใช้ข้อมูลอ้างอิง 1920s ของจีน"

```

## 📦 การขยายคลังความรู้

### การเพิ่มคลังความรู้ใหม่

1. **ระบุประเภท**: genres / craft / references


2. **สร้างไฟล์**:



```bash
   # ความรู้ประเภทนิยาย
   touch templates/knowledge-base/genres/sci-fi.md

   # เทคนิคการเขียน
   touch templates/knowledge-base/craft/worldbuilding.md

   # ข้อมูลอ้างอิง
   mkdir -p templates/knowledge-base/references/ancient-rome
   touch templates/knowledge-base/references/ancient-rome/military.md

```

3. **อัปเดตตารางจับคู่**: เพิ่มรายการที่เกี่ยวข้องลงในตารางจับคู่คำสำคัญของ README.md นี้


4. **เขียนเนื้อหา**: อ้างอิงตามรูปแบบมาตรฐานของคลังความรู้ที่มีอยู่ (ประมาณ 500-800 บรรทัด)



### รูปแบบมาตรฐานของคลังความรู้

ไฟล์คลังความรู้แต่ละไฟล์ควรประกอบด้วย:

```markdown
# [ชื่อหัวข้อคลังความรู้]

## การอ้างอิงด่วน (Quick Reference)
[บทสรุปภาพรวม 1-3 ย่อหน้า อธิบายหลักการสำคัญ]

## หลักการหลัก (Core Principles)
[กฎสำคัญ 3-5 ข้อ พร้อมคำอธิบายโดยละเอียด]

## การประยุกต์ใช้ในทางปฏิบัติ (Practical Application)
[วิธีการนำความรู้เหล่านี้ไปใช้ในแต่ละขั้นตอนของการสร้างสรรค์ผลงาน]

## หลุมพรางที่พบบ่อย (Common Pitfalls)
[ข้อผิดพลาดที่ผู้เขียนมือใหม่มักจะทำและวิธีหลีกเลี่ยง]

## การวิเคราะห์ตัวอย่าง (Examples)
[การวิเคราะห์กรณีศึกษาจากผลงานระดับคลาสสิก]

## การบูรณาการร่วมกับคำสั่งต่างๆ (Integration with Commands)
[วิธีการประยุกต์ใช้ในคำสั่ง /specify, /plan, /write, /analyze]

```
