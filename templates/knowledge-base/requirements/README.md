# คลังความรู้ข้อกำหนดการเขียน (Writing Requirements Knowledge Base)

สารบัญนี้ประกอบด้วยข้อกำหนดการเขียน 8 รูปแบบ ซึ่งจะถูกตรวจจับและโหลดโดยอัตโนมัติผ่านสกิล **requirement-detector**

---

## 📋 ข้อกำหนดที่พร้อมใช้งาน

### 1. anti-ai-v4 (ลดกลิ่นอาย AI เวอร์ชัน 4.0 - แบบสุดโต่ง)

**บริบทที่เหมาะสม**: สำหรับผู้เขียนที่อ่อนไหวต่อกลิ่นอายของ AI เป็นพิเศษ

**คุณลักษณะเด่น**:
- แนวทางการลดกลิ่นอาย AI ที่ดุดันที่สุด
- แบนคำศัพท์มากกว่า 200+ คำ
- แบนโครงสร้างประโยคแบบ AI หลากหลายรูปแบบ
- อาจต้องแลกด้วยการสูญเสียอรรถรสทางวรรณศิลป์ไปบ้าง

**คำสำคัญในการเปิดใช้งาน**:

```

กลิ่นอาย AI แรงเกินไป, ดูเหมือนหุ่นยนต์เขียน, อย่าใช้แนวการเขียนแบบ AI
เหมือนคนแต่ง, มีความเป็นมนุษย์, v4.0, เวอร์ชันสุดโต่ง
ตัดคำสุภาษิตออก, ตัดคำสี่พยางค์ออก, ลด AI ขั้นสุด

```

---

### 2. anti-ai-v3 (ลดกลิ่นอาย AI เวอร์ชัน 3.1 - แบบมาตรฐาน)

**บริบทที่เหมาะสม**: รักษาสมดุลระหว่างการลดกลิ่นอาย AI และคุณค่าทางวรรณศิลป์

**คุณลักษณะเด่น**:
- แนวทางการลดกลิ่นอาย AI ระดับมาตรฐาน
- คงเหลือภาษาเขียนที่จำเป็นไว้
- มีความสมดุลดีที่สุด

**คำสำคัญในการเปิดใช้งาน**:

```

ลดกลิ่นอาย AI, อย่าให้ดูเหมือนหุ่นยนต์เขียน, v3.1, เวอร์ชันมาตรฐาน
เป็นธรรมชาติหน่อย, สมจริง

```

---

### 3. fast-paced (แนวตบหน้า/เดินเรื่องเร็ว)

**บริบทที่เหมาะสม**: นวนิยายแนวแฟนตาซี, นิยายเมือง (Urban Fantasy), นิยายดำเนินเรื่องเร็ว

**คุณลักษณะเด่น**:
- ต้องมีจุดสะใจ (จุดเด่น) อย่างน้อย 2 จุดในทุกบท
- ปมขัดแย้งหนาแน่น
- ห้ามยืดเนื้อหาหรือดึงจังหวะ
- มีเบ็ดเกี่ยวคนอ่าน (Cliffhanger) ทุกบท

**คำสำคัญในการเปิดใช้งาน**:

```

นิยายเน้นความสะใจ, เดินเรื่องเร็ว, จังหวะกระชับ, ไม่ยืดเยื้อ
ต้องมีจุดสะใจทุกบท, ตบหน้า, โชว์เหนือ/เทพ

```

---

### 4. no-poison (แนวไร้พิษ/พล็อตไม่ออกทะเล)

**บริบทที่เหมาะสม**: นวนิยายที่ค่านิยมถูกต้อง มีตรรกะและเหตุผลรองรับอย่างสมเหตุสมผล

**คุณลักษณะเด่น**:
- หลีกเลี่ยงการลดสติปัญญาของตัวเอก (ไอคิวลด)
- หลีกเลี่ยงการสร้างความเข้าใจผิดแบบฝืนพล็อต
- หลีกเลี่ยงบทบาทประเภท "แม่พระ/พ่อพระ" จนเกินพอดี (Saint Mary)
- ตัวละครสมทบต้องมีแรงจูงใจที่สมเหตุสมผล

**คำสำคัญในการเปิดใช้งาน**:

```

ไม่มีจุดพิษ (ไร้พล็อตขัดใจ), อย่ามีจุดพิษ, ค่านิยมถูกต้อง
อย่าลดไอคิวตัวละคร, อย่าฝืนพล็อต, ตรรกะสมเหตุสมผล

```

---

### 5. serious-literature (แนววรรณกรรมสร้างสรรค์/วรรณกรรมจริงจัง)

**บริบทที่เหมาะสม**: ผลงานที่เน้นคุณค่าทางวรรณกรรมและความลึกซึ้ง

**คุณลักษณะเด่น**:
- มุ่งเน้นไปที่แง่คิดและปรัชญา
- การถ่ายทอดลักษณะตัวละครอย่างลึกซึ้ง
- ใช้กลวิธีการเขียนแบบสัจนิยม (Realism)
- ควบคุมเนื้อหาเชิงบันเทิงให้อยู่ในระดับที่พอดี

**คำสำคัญในการเปิดใช้งาน**:

```

วรรณกรรมจริงจัง, วรรณกรรมบริสุทธิ์, มีความลึกซึ้ง
สัจนิยม, มีแง่คิด, สำรวจความเป็นมนุษย์

```

---

### 6. strong-emotion (แนวอารมณ์เข้มข้น)

**บริบทที่เหมาะสม**: นวนิยายที่เน้นการขับเคลื่อนด้วยอารมณ์อันแรงกล้า

**คุณลักษณะเด่น**:
- อารมณ์อัดแน่นเต็มพิกัด
- ความขัดแย้งรุนแรงสะเทือนใจ
- จุดระเบิดทางอารมณ์เด่นชัด
- ดำเนินเนื้อหาจากจุดไคลแมกซ์สู่จุดไคลแมกซ์อย่างต่อเนื่อง

**คำสำคัญในการเปิดใช้งาน**:

```

อารมณ์รุนแรง, อารมณ์เข้มข้น, อารมณ์อัดแน่น
ไคลแมกซ์ต่อเนื่อง, ระเบิดอารมณ์

```

---

### 7. romance-sweet (แนวนิยายรักสายสุขนิยม/ฟีลกู๊ด)

**บริบทที่เหมาะสม**: นวนิยายรักแนวโรแมนติกคอมเมดี้/สายหวาน

**คุณลักษณะเด่น**:
- สาดความหวานเต็มที่
- ความขัดแย้งมีแต่พอเหมาะ
- หลีกเลี่ยงเนื้อหาที่ทำร้ายจิตใจ (ดราม่าปวดตับ)
- จบแบบ Happy Ending (HE)

**คำสำคัญในการเปิดใช้งาน**:

```

นิยายสายหวาน, หวานเจี๊ยบ, โรแมนติก, หวานฟิน
ไม่ดราม่า, ขอหวานๆ, แจกความหวาน

```

---

### 8. romance-angst (แนวนิยายรักสายดราม่า/ตับพัง)

**บริบทที่เหมาะสม**: นวนิยายรักแนวเรียกน้ำตา/ดราม่าหนักหน่วง

**คุณลักษณะเด่น**:
- การทำร้ายจิตใจตัวละครต้องมีเหตุผลรองรับ
- ต้องแสดงให้เห็นถึงกระบวนการและความเจ็บปวด
- ความดราม่าต้องมีขอบเขตที่จำกัด
- สุดท้ายต้องมีการปรับความเข้าใจหรือคลี่คลาย

**คำสำคัญในการเปิดใช้งาน**:

```

นิยายตับพัง, ดราม่าปวดตับ, แจกเศษแก้ว/ใบมีด, แนว Bad Ending (BE)
ขอแบบดราม่า, เจ็บปวดสะใจ

```

## 🔄 กลไกการทำงาน

### ขั้นตอนการตรวจจับอัตโนมัติ

1. **ดักฟังข้อมูลนำเข้าของผู้ใช้**
   - ตรวจสอบภายในข้อกำหนด (specification), กฎเกณฑ์ (constitution) และบทสนทนา
   - ระบุคำสำคัญ (Keywords) ของข้อกำหนด

2. **เปิดใช้งาน requirement-detector**
   - วิเคราะห์คำสำคัญ
   - จับคู่กับข้อกำหนดที่ตรงกัน

3. **โหลดคลังความรู้**
   - อ่านไฟล์ข้อกำหนดที่เกี่ยวข้อง
   - แสดงกฎเกณฑ์หลัก

4. **บันทึกการตั้งค่า**
   - ทำเครื่องหมายไว้ใน `specification.md`
   - นำไปใช้โดยอัตโนมัติเมื่อมีการเรียกคำสั่ง `/write`

## 🎯 การใช้งานร่วมกัน

### ชุดผสมที่แนะนำ

**นิยายเมืองแนวตบหน้า (Urban Fast-paced)**:

```

writing-style: natural-voice
writing-requirements:

* fast-paced
* anti-ai-v4
* no-poison

```

**วรรณกรรมจริงจัง (Serious Literature)**:

```

writing-style: literary
writing-requirements:

* serious-literature
* anti-ai-v3

```

**นิยายรักสายฟีลกู๊ด (Sweet Romance)**:

```

writing-style: natural-voice
writing-requirements:

* romance-sweet
* anti-ai-v4

```

**นิยายรักสายปวดตับ (Angst Romance)**:

```

writing-style: literary
writing-requirements:

* romance-angst
* strong-emotion

```

---

## ⚠️ การตรวจจับข้อกำหนดที่ขัดแย้งกัน

ข้อกำหนดบางประการอาจขัดแย้งกันเอง ซึ่งระบบจะแจ้งเตือนโดยอัตโนมัติ:

**ตัวอย่างความขัดแย้งที่ 1**:

```

fast-paced (เดินเรื่องเร็ว) + serious-literature (วรรณกรรมจริงจัง)
→ ขัดแย้งกัน: วรรณกรรมจริงจังจำเป็นต้องมีการปูเนื้อเรื่องอย่างค่อยเป็นค่อยไป (ดำเนินเรื่องช้า)

```

**ตัวอย่างความขัดแย้งที่ 2**:

```

romance-sweet (สายหวาน) + romance-angst (สายปวดตับ)
→ ขัดแย้งกัน: ความหวานและความดราม่าปวดตับเป็นสิ่งที่ตรงข้ามกัน

```
